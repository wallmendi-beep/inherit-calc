# 업데이트 노트 (v4.32) - 사용자 조작 권한 강화 및 초기 UX 최적화

본 버전에서는 사망자의 사망 시차(선사망 vs 후사망)에 따른 시스템의 과도한 개입을 바로잡고, 앱 진입 시 사용자가 직관적으로 다음 단계를 파악할 수 있도록 초기 안내 로직을 전면 개편했습니다.

## 1. 주요 변경 사항

### 🛠️ 사망 시차별 토글 지형도 고도화
*   **후사망자(재상속) 자동 ON 전환**: 사망 일자 입력 시 '피상속인 사후 사망'으로 판별되면 상위 탭에서 즉시 상속 제외 상태가 해제(`isExcluded: false`)됩니다. 이를 통해 재상속 데이터를 입력하기 위해 토글을 수동으로 조작해야 했던 번거로움을 제거했습니다.
*   **선사망자(대습상속) 가드레일 유지**: 선사망자의 경우 데이터 없이 토글이 켜지는 것을 방지하는 '3초 자동 원복 타이머'를 유지하여 법리적 정합성과 UI 보호 기능을 보존했습니다.
*   **엔진 전처리 필터 보정**: 하위 데이터가 없는 후사망자를 시스템이 '제외 대상'으로 오판하여 토글을 강제로 끄던 로직을 제거하여 재상속 데이터 입력의 흐름을 확보했습니다.

### ✨ 초기 UX 및 스마트 가이드 최계화
*   **입문 가이드 최우선 노출**: 피상속인의 [성명] 또는 [사망일자]가 없는 초기 상태에서는 복잡한 상속 분배 안내를 배제하고 **"사건번호와 피상속인의 기본정보를 입력해주세요."**라는 메시지를 단독 노출합니다.
*   **중앙 브리핑 영역 정제**: 화면 중앙 안내 상자에서도 기초 정보 부재 시 "형제자매 분배" 등 혼란스러운 설명을 차단하고, 입문 단계에 맞는 안내 문구로 대체했습니다.

### 📜 디자인 룰(SSOT) 동기화
*   `디자인 룰/main_display_rules.md` 개정:
    - **제6조**: 사망 시차별 토글 기본값 및 가드레일 규칙 명문화.
    - **제7조**: 초기 데이터 공백 시 입문 가이드 최우선 순위 규정 신설.

## 2. 기술적 수정 내역
*   `src/utils/treeDomain.js`: `updateDeathInfo` 내 후사망자 자동 ON 로직 추가.
*   `src/hooks/useSmartGuide.js`: 기초 정보 누락 시 가이트 필터링 및 전용 메시지 리턴 로직 적용.
*   `src/components/InputPanel.jsx`: 초기 상태 브리핑 영역 렌더링 분기 처리.
*   `src/App.jsx`: `preprocessTree` 엔진의 후사망자 강제 제외 조건 삭제.

## 3. 검증 결과
*   선사망자 입력 시: 3초 가드레일 정상 작동 확인.
*   후사망자 날짜 변경 시: 토글 자동 ON 및 지분 즉시 노출 확인.
*   초기화 상태: "사건번호와 피상속인 정보 입력" 가이드 정상 노출 확인.
