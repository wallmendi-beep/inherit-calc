# 상속지분 계산기 입력창 표시 및 안내 규칙 (v1.5)

정보 입력창(InputPanel) 및 안내 시스템의 **일관성**, **실시간 반응**, **예우(UX)**를 확보하기 위한 기술 규칙입니다.

---

## 1. 명칭 및 호칭 규칙

### 1.1. 고정 범위 (Fixed Scope)
*   **자녀**: 아들, 딸로 구분합니다.
*   **배우자**: 처, 남편으로 구분합니다.

### 1.2. 존칭 배제 (Strict No-Honorifics)
*   입력 정보 및 시스템 명칭에서 **"님" 존칭은 일괄 배제**합니다. (예: `[이름]님` -> `[이름]`)

### 1.3. 계보별 명칭 구분 (Generational Naming)
부, 모 등 탭별 그룹 명칭 및 안내 문구는 **상속 관계**를 기준으로 명명합니다.
*   **[최상위]**: 최상단 Root 피상속인, 또는 **피상속인 (상속 개시 대상자)**
*   **[하위]**: **피대습상속인 (선사망자 < 피상속인)**

---

## 2. Vault 기반 다계층 상태 관리 (Context-aware)

### 2.1. 탭별 독립적 상태 판별
한 인물이 다중 상속 관계를 맺을 때(부, 모 양방향 상속 등), 각 탭의 피상속인 사망 시점과 상속인의 사실 관계를 대조하여 상태를 독립적으로 표시합니다.

*   **상태 토글(동일/출가/호주)**: 
    - 상속 개시일(피상속인 사망일)과 상속인의 혼인일/호주승계일을 대조하여 자동 반영합니다.
    - 예: 부(父) 사망 시에는 출가녀이나, 모(母) 사망 시(혼인 전인 경우 등)에는 동일가적으로 표시될 수 있습니다.

### 2.2. 선사망자(Predeceased) 처리
*   피상속인보다 먼저 사망한 상속인은 **`[상속권 없음 (선사망)]`** 뱃지를 고정 표시합니다.
*   선사망자의 경우 상속 포기, 결격 등의 옵션 선택 드롭다운을 차단하고 텍스트로 고지합니다.

### 2.3. 상실선고 시계열 필터
*   **2024년 4월 25일** 이후 상속 개시 사건에서만 '상실선고' 옵션을 활성화합니다.

## 3. 스마트 가이드 및 안내 문구

상황별로 입력이 누락되었거나 주의가 필요한 지점을 명확하게 안내합니다.

| 상황 | 안내 문구 예시 (실시간 피드백) |
| :--- | :--- |
| **선사망 존재** | `[이름]은 피상속인보다 먼저 사망하였습니다. 대습상속(자녀/배우자) 데이터가 있다면 입력해 주세요.` |
| **계보 전이** | [이름]의 직계비속이 없어, **민법 제1000조**에 따라 차순위인 **[직계존속]**에게 상속권이 이전됩니다. |
| **상속 포기** | [이름]의 상속포기로 인해, 해당 지분은 **[이름1], [이름2]**에게 균등 배분됩니다. |

## 4. 레이아웃 및 시각적 표현 규칙

### 4.1. 상속인 행(HeirRow) 정렬 수치 (Fixed Margins)
앱의 수평 정렬을 유지하기 위해 아래 수치를 절대적으로 준수합니다.
*   **드래그 핸들**: `ml-[10px]`
*   **상태 토글**: `ml-[20px]`
*   **성명 입력**: `ml-[50px]`
*   **관계 선택**: `ml-[30px]`
*   **사망여부**: `ml-[30px]`
*   **특수조건**: `ml-[10px]`
*   **삭제 버튼**: `ml-[30px] mr-[20px]`

### 4.2. 뱃지 및 안내창
*   상태 변화가 일어나는 배율은 뱃지`[]` 형태를 사용하며, 감산 시 **Rose (#c93f3a 계열)** 테두리를 적용합니다.
*   선사망자 발생 시 행 하단에 **인라인 슬라이드 경고 배너**를 표시하여 사용자 주의를 환기합니다. (배우자 상시 노출, 그 외 상속인은 토글 On 시 즉각 노출 후 3초 뒤 자동 Off 원복 및 슬라이드 삭제 처리)

### 4.3. 지능형 자동 판별 및 메모리 UI
*   **출가/동일가적 토글**: 시스템에 의해 자동 판별된 상태는 **블루톤 스타일(`border-blue-400`, `bg-blue-50/50`)**을 적용하며, 마우스 호버 시 **"자동판별"** 툴팁을 표시합니다. (✨ 아이콘은 사용하지 않음)
*   **날짜 메모리(Memory)**: 날짜 입력 후 삭제하더라도 내부적으로 백업(`_lastDate`)하며, 비어있는 입력란 옆에 **[복원]** 버튼을 노출하여 사용자 실수를 방지합니다.
*   **가이드 통합**: 스마트 가이드와 데이터 감사(Audit) 항목은 중복을 제거하여 하나의 단일 리스트로 가이드 패널에 표시합니다.

## 6. 사망 시차에 따른 자동 분배 추정 및 가이드 규칙 (v1.7)

## 제6조 (상속인의 사망 시차별 가이드 및 토글 규칙)

대습상속(선사망)과 재상속(후사망)의 시각적 질서를 위해 아래 규칙을 엄격히 적용한다.

1. **선사망 (대습상속)**: 피상속인보다 먼저 사망한 경우
   - **기본 상태**: [상속에서 제외(Off)]를 기본값으로 한다.
   - **가드레일**: 사용자가 토글을 ON으로 변경하면 **3초간 시각적 ON 상태**를 유지하며 대습상속 데이터 입력을 유도한다. 하위에 상속인 데이터가 추가되지 않으면 3초 후 다시 자동으로 OFF(제외) 처리된다.
   - **안내문**: "대습상속인을 입력하지 않으면 계산에서 제외됩니다"라는 슬라이드 경고를 노출한다.

2. **후사망 (재상속)**: 피상속인보다 나중에 사망한 경우
   - **기본 상태**: **[상속인 포함(On)]을 기본값**으로 하며, 본래의 상속지분(1분의 1 등)을 즉시 노출한다.
   - **지능형 전환**: 사망일 입력 시 후사망으로 판별되면 시스템은 즉시 상속 제외 상태를 해제(isExcluded: false)해야 한다.
   - **재상속 가이드**: 별도의 상속인 입력이 없으면 직계존속 혹은 형제자매 중 생존자에게 지분이 분배됨을 실명으로 안내한다.

## 제7조 (초기 상태 안내 및 가계도 진입 규칙)

사용자가 앱에 처음 진입했거나 데이터를 초기화했을 때, 시스템은 전문가적 법리보다 **기초 데이터의 완결성**을 우선적으로 가이드한다.

1. **초기 안내 메시지 우선순위**:
   - 피상속인의 **[성명]** 또는 **[사망일자]**가 입력되지 않은 경우, 어떠한 상속 지분 전이 가이드나 경고문보다 **"사건번호와 피상속인의 기본정보를 입력해주세요."**라는 안내를 최우선으로 단독 노출한다.
   - 사건번호(`caseNo`)가 누락된 경우에도 상단 툴바나 가이드 라인을 통해 입력을 독려한다.

2. **지능형 가이드 억제**:
   - 기초 정보 누락 상태에서는 "상속인이 없습니다", "형제자매에게 분배됩니다" 등 사용자에게 혼란을 줄 수 있는 자동 추정 로직 노출을 억제한다.

3. **가계도 진입 시점**:
   - 최소한 피상속인의 이름과 사망일이 입력된 시점부터 전체 가계도 분석 및 스마트 가이드 엔진을 본격 가동한다.
