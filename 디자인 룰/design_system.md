# 🎨 상속지분 계산기 PRO v4.28 디자인 시스템 (UDS)

이 문서는 앱의 시각적 일관성을 유지하고 '전문적 신뢰감'을 확보하기 위한 최종 디자인 헌법입니다.

## 0. 디자인 정체성 (Identity)
본 앱은 **Notion-inspired Professional Minimalist** 스타일을 지향합니다. 구글 머티리얼 디자인의 화려함보다는 종이 질감의 무채색과 정갈한 타이포그래피를 통한 신뢰도 높은 데이터 전달을 최우선으로 합니다.

## 1. 전역 컬러 팔레트 (Core Palette)

모든 색상은 아래 정의된 HEX 코드를 엄격히 준수하며, 임의의 투명도나 유사 색상 사용을 지양합니다.

| 구분 | HEX 코드 | 명칭 | 용도 |
| :--- | :--- | :--- | :--- |
| **Black** | `#37352f` | Notion Black | 주요 본문 텍스트명, 피상속인 성명 |
| **Gray** | `#787774` | Notion Gray | 보조 설명, 메타 정보, 비활성 텍스트 |
| **Navy** | `#1e56a0` | Deep Navy | **핵심 데이터(상속지분)**, 활성 강조 포인트 |
| **Blue** | `#0284c7` | Sky Blue | 액션 버튼(호적연혁), 인계 탭 링크 |
| **Red** | `#c93f3a` | Muted Red | 직접적인 오류, 상속권 상실 등 크리티컬 경고 |
| **Brown** | `#92400e` | Amber Brown | 법적 제약 정중한 안내, 차분한 경고 |

## 2. 타이포그래피 표준 (Typography Tiers)

시스템 폰트는 **Pretendard**를 우선하며, 계층별 크기와 두께를 고정합니다.

- **L1 (Heading)**: `17px / Bold / #37352f` - 섹션 제목, 모달 타이틀
- **L2 (Body/Input)**: `15px / Medium / #37352f` - 상속인 성명 입력, 주요 본문
- **L3 (Label/Small)**: `13px / Semibold / #787774` - 설명 문구, 미니 레이블
- **L4 (Data)**: `11px / Black / #1e56a0` - 지분율(N/D) 수치 (가장 선명하게)

## 3. 컴포넌트 규격 (Components)

### 3.1. 버튼 (Buttons)
- **곡률**: `rounded-md` (0.375rem) 고정.
- **그림자**: `shadow-sm`, 호버 시 `shadow`로 시각적 반응.

### 3.2. 알림 및 안내 (Feedback)
- **일반 경고**: 가로로 긴 팝업보다는 해당 영역 하단에서 **슬라이드 다운(`animate-in`)**되는 인라인 방식을 권장합니다.
- **어조**: "수정해 주세요", "확인해 주세요" 등 객관적이고 정중한 표현을 사용합니다.
