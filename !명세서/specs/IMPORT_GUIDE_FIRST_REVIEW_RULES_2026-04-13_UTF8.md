# 불러오기 후 즉시 검수 가이드 규칙

## 목적
- AI/파일 불러오기에서 구조 오류가 의심되더라도 사용자의 진행을 막지 않는다.
- 데이터를 먼저 불러온 뒤, 입력 화면과 스마트 가이드에서 직접 수정하도록 유도한다.

## 기본 원칙
- `confirm`으로 진행 여부를 묻지 않는다.
- 불러오기 선검사 결과는 `importIssues`로 저장한다.
- `importIssues`는 스마트 가이드 항목으로 노출한다.
- 문제가 있으면 기본 이동 탭은 `calc`가 아니라 `input`으로 둔다.

## 적용 대상
- 파일 JSON 불러오기
- AI JSON 입력

## 선검사 항목 예시
- 이름 누락
- 관계값 이상
- personId 중복
- 사망일 누락
- 사망자인데 하위상속인 비어 있음
- 유효 배우자 중복

## UX 규칙
- 불러오기 자체는 계속 진행한다.
- 검수 항목은 사람별 가이드 카드로 뜬다.
- 가이드 문구는 `불러오기 직후 입력값을 확인하고 저장한 뒤 계속 진행해 주세요.`를 공통 후행 문구로 둔다.
- 검수 대상이 있으면 입력 탭으로 보내 즉시 수정 가능하게 한다.

## 상태 갱신 규칙
- `importIssues`는 불러오기 직후 설정한다.
- 사용자가 입력을 수정하면 현재 트리를 다시 검사하여 `importIssues`를 자동 갱신한다.
- 문제가 모두 해소되면 `importIssues`는 빈 배열이 되고, 해당 가이드도 사라진다.

## 재발 방지 원칙
- 불러오기 검수는 차단형 모달보다 비차단형 가이드가 우선이다.
- 사용자가 이미 앱 안으로 들어온 뒤에는 “취소 후 다시 입력”보다 “바로 수정”이 우선이다.
