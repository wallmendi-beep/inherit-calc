# 불러오기 선검사 및 재입력 유도 규칙

## 목적
- AI/파일 불러오기 직후, 계산 단계로 넘어가기 전에 구조상 의심되는 오류를 먼저 발견한다.
- 사용자가 잘못된 JSON을 그대로 진행하지 않도록 재입력을 유도한다.

## 기본 원칙
1. JSON 파싱 성공만으로 정상 입력으로 보지 않는다.
2. `normalizeImportedTree()` 후의 구조를 기준으로 선검사를 수행한다.
3. 확인이 필요한 항목이 있으면 즉시 경고하고,
   사용자가 원하면 계속 불러오고, 아니면 재입력하도록 멈춘다.

## 선검사 대상
- 이름이 비어 있는 상속인
- 관계값이 유효하지 않은 상속인
- personId 중복
- 사망으로 입력되었는데 사망일이 없음
- 사망자인데 하위상속인이 비어 있음
  - 단, 선사망 배우자는 예외
- 동일 단계에 유효 배우자가 2인 이상 존재

## UX 규칙
- 경고 메시지는 최대 6건까지 목록으로 보여준다.
- 문구는 “다시 입력을 권장합니다. 그래도 불러오시겠습니까?”로 마무리한다.
- 사용자가 취소하면 현재 입력 상태를 유지하고 불러오기를 중단한다.
