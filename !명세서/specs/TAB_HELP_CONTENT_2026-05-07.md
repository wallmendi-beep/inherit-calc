관련 문서: `TAB_STRUCTURE_RULES.md`, `CORE_LOGIC_SPECS.md`, `FLOW_TABLE_JUDGMENT_LABEL_RULES_2026-05-07.md`, `PREDECEASE_REINHERITANCE_JUDGMENT_AND_WARP_RULES.md`

## 1. 작성 원칙

이 문서는 화면 도움말 버튼에 들어갈 사용자용 설명 원문이다.

원칙:

- 도움말은 기능 설명보다 사용자가 지금 무엇을 확인해야 하는지 알려준다.
- 계산 법리를 길게 강의하지 않고, 현재 화면의 역할과 다음 행동을 설명한다.
- 같은 내용을 여러 탭에서 반복하지 않는다.
- 입력 탭은 사실관계 수정, 사건 검토 탭은 사건별 분배·판정 검토, 상속지분 탭은 최종 상속인별 수렴·검산을 맡는다.
- 판정 로직 도움말은 상세 법리 문서가 아니라 앱이 어떤 기준으로 계산하는지 이해시키는 요약이어야 한다.

## 2. 데이터 입력 탭 도움말

### 2.1 도움말 제목

```text
데이터 입력 탭 사용법
```

### 2.2 한 줄 설명

```text
이 탭은 상속지분 계산의 원천이 되는 가족관계와 사건 사실을 입력하고 수정하는 곳입니다.
```

### 2.3 본문 원문

데이터 입력 탭에서는 계산 결과를 보는 것이 아니라, 계산에 사용할 사실관계를 정리합니다.  
이 탭에서 입력한 내용이 사건 검토 탭과 상속지분 탭의 기준이 됩니다.

먼저 피상속인의 기본정보를 확인합니다.

- 사건번호
- 피상속인 성명
- 사망일자
- 최초 분배 지분

그 다음 상속인별 정보를 확인합니다.

- 성명
- 피상속인과의 관계
- 생존 또는 사망 여부
- 사망일자
- 혼인일, 재혼일, 이혼일, 복적일
- 성별
- 호주 여부
- 동일가적 또는 비동일가적
- 상속권 없음 또는 제외 사유

사망한 상속인이 있으면 그 사람의 하위 상속인도 확인해야 합니다. 하위 상속인을 입력하지 않은 상태가 실제로 “상속인이 없음”인지, 아직 입력하지 않은 것인지 앱은 자동으로 확정하지 않습니다.

필요한 하위 상속인이 있으면 상속인을 추가합니다. 더 입력할 사람이 없으면 상황에 맞게 `추가 상속인 없음`, `대습상속인 없음` 같은 확정 버튼을 사용합니다. 확정하지 않으면 스마트 가이드나 경고가 계속 남을 수 있습니다.

AI 입력이나 파일 불러오기는 완성된 계산 결과가 아니라 입력 초안입니다. 이름, 관계, 사망일, 배우자, 하위 상속인 누락 여부를 반드시 확인해야 합니다. 사용자가 직접 수정한 값이 AI가 만든 값보다 우선합니다.

좌측 가계도 요약은 전체 가족 구조를 빠르게 확인하기 위한 보조 패널입니다. 특정 사건의 분배 결과나 판정 이유는 사건 검토 탭에서 확인합니다.

### 2.4 AI 입력 방법과 주의사항

AI 입력은 가족관계 자료를 빠르게 입력하기 위한 보조 기능입니다. AI가 만든 결과는 확정된 계산 자료가 아니라 사용자가 검토해야 할 초안입니다.

사용 방법:

1. 가족관계 자료나 정리된 사실관계를 준비합니다.
2. AI 입력 기능에 자료를 넣어 JSON 초안을 생성합니다.
3. 앱은 AI 결과에서 허용된 필드만 남기고 정규화합니다.
4. 정규화된 결과가 데이터 입력 탭에 들어오면, 사용자가 직접 내용을 확인합니다.
5. 스마트 가이드와 경고가 표시되면 해당 사람 또는 사건으로 이동해 수정합니다.

AI 입력 후 반드시 확인할 항목:

- 이름이 누락되거나 잘못 읽힌 사람이 없는가
- 관계가 부모·자녀·배우자 등으로 맞게 들어갔는가
- 사망자인데 사망일자가 빠진 사람이 없는가
- 사망자 아래 하위 상속인이 누락되지 않았는가
- 같은 단계에 배우자가 중복으로 들어가지 않았는가
- 후혼 배우자나 후혼 자녀가 특정 사건의 상속인 범위에 잘못 섞이지 않았는가
- 구법 사건에서 혼인일, 복적일, 동일가적·비동일가적, 호주 여부가 빠지지 않았는가

주의사항:

- AI 입력 결과를 그대로 계산 확정으로 보지 않는다.
- AI는 구조 오류를 스스로 해결하지 않는다.
- AI가 만든 이름·관계·날짜보다 사용자가 수정한 값이 우선한다.
- 원문에는 존재하지만 특정 사망 사건의 상속인이 아닌 사람이 함께 들어올 수 있다.
- 하위 상속인이 비어 있는 사망자는 실제 무후인지, AI가 누락한 것인지 사용자가 확인해야 한다.
- 경고가 뜬 상태에서 계산 결과가 나오더라도 실무 적용 전 반드시 입력 탭에서 확인한다.

AI 입력은 “자동 계산” 기능이 아니라 “입력 초안 생성 + 사용자 검토” 기능이다.

### 2.5 확인 체크리스트

- 피상속인의 성명과 사망일자가 입력되어 있는가
- 각 상속인의 관계가 맞는가
- 사망한 사람의 사망일자가 입력되어 있는가
- 사망한 사람 아래에 필요한 후속 상속인이 입력되어 있는가
- 실제로 후속 상속인이 없다면 없음 확정을 했는가
- 구법 사건에서 호주 여부와 동일가적·비동일가적 상태를 확인했는가
- AI 또는 파일 불러오기 후 누락·중복·관계 오류가 없는지 검토했는가

### 2.6 짧은 툴팁 문구

```text
가족관계와 사건 사실을 입력하는 곳입니다. 계산 결과가 이상하면 먼저 이 탭의 관계, 날짜, 후속 상속인 입력을 확인하세요.
```

## 3. 사건 검토 탭 도움말

### 3.1 도움말 제목

```text
사건 검토 탭 사용법
```

### 3.2 한 줄 설명

```text
이 탭은 피상속인별 사망 사건마다 상속지분이 어떻게 분배되고 이어지는지 검토하는 곳입니다.
```

### 3.3 본문 원문

사건 검토 탭은 입력된 가족관계를 바탕으로 각 사망 사건의 분배 과정을 보여줍니다.  
여기서는 최종 합계보다 “왜 이 사람에게 지분이 갔는지”, “어느 지점에서 재상속이나 대습상속이 열렸는지”를 확인합니다.

화면은 좌측 사건 보고서와 우측 상속 흐름표로 나뉩니다.

좌측 사건 보고서는 현재 선택한 노드가 속한 하나의 상속사건을 설명합니다. 사건 관계인, 사건 요약, 판정 요약을 통해 해당 사건에서 누가 상속인이 되었고 누가 제외되었는지 확인합니다.

우측 상속 흐름표는 사건별 분배 결과를 표로 보여줍니다. 상속인, 관계, 취득지분, 흐름, 판정 요소를 확인할 수 있습니다.

`제N사건`은 실제 사망 사건의 번호입니다. 한 피상속인이 사망하여 그 사람의 지분을 다시 나누는 계산 단계가 하나의 사건입니다.

`흐름 M`은 새 사건이 아닙니다. 같은 사망 사건으로 들어온 지분 원천이 여러 개인 경우, 화면에서 이해하기 쉽게 원천별로 나누어 보여주는 하위 흐름입니다.

예를 들어 김철수가 홍길동 쪽 지분과 김영희 쪽 지분을 모두 받은 뒤 사망했다면, 둘은 모두 같은 김철수 사건입니다. 다만 지분 원천이 다르므로 화면에서는 `제3사건 · 흐름 1`, `제3사건 · 흐름 2`처럼 나누어 보일 수 있습니다.

흐름 칸은 지분의 이동 상태를 표시합니다.

- 최종 생존 상속인에게 귀속되면 `최종 취득`
- 후사망자가 받은 지분이 다시 그 사람의 상속인에게 넘어가면 `재상속`
- 선사망자 계통으로 지분이 이어지면 `대습상속`
- 상속권이 없으면 `상속권 없음`

판정 요소 칸은 지분이 달라진 이유를 짧게 보여줍니다. 예를 들어 배우자 가산, 호주상속 가산, 비동일가적 감산, 상속권 없음 사유 등이 표시됩니다.

사건 검토 탭에서 이상한 결과가 보이면 바로 계산식을 고치는 것이 아니라, 데이터 입력 탭에서 관계·날짜·후속 상속인 입력을 확인해야 합니다.

### 3.4 사건번호와 흐름 번호 설명

| 표시 | 의미 |
|---|---|
| `제1사건`, `제2사건` | 실제 사망 사건의 순서 |
| `제N사건 · 흐름 1` | 제N사건으로 들어온 첫 번째 지분 원천 |
| `제N사건 · 흐름 2` | 제N사건으로 들어온 두 번째 지분 원천 |

도움말 예시:

```text
제3사건 · 흐름 1
홍길동 → 김철수 · 1/15

제3사건 · 흐름 2
김영희 → 김철수 · 2/165
```

두 줄 모두 김철수의 사망 사건이므로 사건번호는 같습니다. 지분이 들어온 원천만 다릅니다.

### 3.5 확인 체크리스트

- 각 사건의 피상속인과 사망일이 맞는가
- 상속인 관계가 사건 기준일에 맞게 표시되는가
- 상속권 없음 사유가 납득되는가
- 배우자 가산, 호주 가산, 구법 감산 표시가 맞는가
- 선사망자는 대습상속 또는 제외로 처리되었는가
- 후사망자는 재상속 사건으로 이어지는가
- 같은 사건이 흐름별로 나뉘어 보여도 실제 사건번호는 유지되는가

### 3.6 짧은 툴팁 문구

```text
사망 사건별로 지분이 어떻게 분배되고 재상속·대습상속으로 이어지는지 확인하는 탭입니다.
```

## 4. 상속지분 탭 도움말

### 4.1 도움말 제목

```text
상속지분 탭 사용법
```

### 4.2 한 줄 설명

```text
이 탭은 사건별로 흩어진 취득 지분을 최종 상속인별로 모아 검산하는 곳입니다.
```

### 4.3 본문 원문

상속지분 탭은 사건 검토 탭에서 계산된 여러 사건의 분배 결과를 최종 상속인별로 모아 보여줍니다.  
사건 검토 탭이 “분배 과정”을 보는 곳이라면, 상속지분 탭은 “최종적으로 누가 얼마나 취득했는지”를 확인하는 곳입니다.

상속지분 탭에는 `취득합산`과 `최종지분표`가 있습니다.

`취득합산`은 기본 보기입니다. 같은 최종 상속인이 여러 경로에서 지분을 받은 경우, 그 지분들을 더해 최종지분으로 수렴합니다.

`경로합산` 칸에는 사람 이름 경로를 쓰지 않고 숫자 합산식만 표시합니다.

```text
① 1/15 + ② 2/165
```

여기서 `①`, `②`는 좌측 취득계보 패널의 `① 경로`, `② 경로`와 대응합니다.

좌측 취득계보 패널은 선택한 상속인의 지분이 누구를 거쳐 내려왔는지를 보여줍니다. 표에서는 숫자만 보고, 경로 설명은 좌측 패널에서 확인합니다.

`최종지분표`는 최종 생존 상속인별 지분을 정리해서 보여줍니다.

- `최종 지분`: 약분된 최종 지분
- `그룹통분 지분`: 같은 재상속·대습 그룹 안에서 비교하기 위한 통분 지분
- `전체통분 지분`: 전체 최종 상속인을 하나의 공통분모로 맞춘 지분

그룹통분 지분은 같은 가족 그룹 안에서 배우자 가산이나 자녀 균분 구조를 눈으로 비교하기 위한 값입니다. 전체통분 지분은 모든 최종 상속인의 지분을 한 분모로 맞춰 전체 합계를 검산하기 위한 값입니다.

상속지분 탭에서 숫자가 이상하면 먼저 취득합산에서 어떤 경로들이 더해졌는지 보고, 필요하면 사건 검토 탭에서 해당 사건의 분배 과정을 확인합니다. 사실관계가 잘못되어 있으면 데이터 입력 탭에서 수정합니다.

### 4.4 취득합산과 최종지분표의 차이

| 보기 | 역할 |
|---|---|
| `취득합산` | 사건별 취득 지분을 최종 상속인별로 더한다 |
| `취득계보` | 선택한 상속인의 각 경로가 누구를 거쳐 왔는지 보여준다 |
| `최종지분표` | 최종 상속인별 지분을 그룹통분·전체통분 기준으로 검산한다 |

### 4.5 확인 체크리스트

- 최종 상속인별 지분이 한 사람당 한 줄로 수렴되는가
- 복수 경로가 있으면 `①`, `②` 번호가 좌측 취득계보와 대응하는가
- 경로합산의 덧셈 결과가 최종지분과 일치하는가
- 같은 가족 그룹 안에서 그룹통분 지분이 같은 분모로 표시되는가
- 전체통분 지분의 합계가 전체 지분과 맞는가
- 숫자가 이상하면 사건 검토 탭의 해당 사건으로 되돌아가 확인했는가

### 4.6 짧은 툴팁 문구

```text
사건별 취득분을 최종 상속인별로 합산하고, 그룹통분·전체통분으로 최종 지분을 검산합니다.
```

## 5. 판정 로직 도움말

### 5.1 도움말 제목

```text
판정 로직 이해하기
```

### 5.2 한 줄 설명

```text
앱은 각 사망 사건의 기준일, 관계, 생사 상태, 가적·호주 정보, 제외 사유를 기준으로 상속권과 지분을 판정합니다.
```

### 5.3 본문 원문

상속지분 계산은 전체 가계도를 한 번에 단순 분배하는 방식이 아닙니다.  
앱은 피상속인이 사망한 사건을 하나씩 만들고, 그 사건의 기준일에서 상속인을 다시 판단합니다.

가장 먼저 사망일을 기준으로 적용 법 시기를 정합니다.

- 1960년 제정 민법 시기
- 1979년 개정 민법 시기
- 1991년 이후 현행 민법 시기

법 시기에 따라 배우자 지분, 여성 상속인의 감산 여부, 호주상속 가산 여부가 달라질 수 있습니다.

그 다음 현재 사건 기준일에서 각 사람의 상태를 판단합니다.

- 생존 상속인인지
- 피상속인보다 먼저 사망한 선사망자인지
- 피상속인보다 나중에 사망한 후사망자인지
- 상속포기, 결격, 상속권 상실 등 제외 사유가 있는지
- 재혼이나 이혼으로 상속권이 없는지
- 구법상 동일가적·비동일가적 여부가 지분에 영향을 주는지
- 호주상속 가산이 적용되는지

선사망자는 현재 사건에서 직접 최종 취득자가 되지 않습니다. 다만 그 사람에게 유효한 대습상속인이 있으면 대습상속 흐름이 열릴 수 있습니다. 유효한 대습상속인이 없으면 그 계통은 분배에서 제외될 수 있습니다.

후사망자는 현재 사건에서 일단 자기 지분을 취득합니다. 그 뒤 그 사람이 다시 사망한 사건이 열리면, 그 지분은 후사망자의 상속인에게 재상속됩니다.

같은 사람이 여러 사건에 반복해서 등장할 수 있습니다. 이것은 오류가 아닙니다. 같은 사람이라도 어느 사건 기준일에서 보느냐에 따라 생존 상속인, 선사망자, 후사망자, 재상속 피상속인이 될 수 있습니다.

`판정 요소`는 계산 결과의 이유를 짧게 보여주는 칸입니다. 예를 들어 다음과 같은 표현이 표시됩니다.

- `균분`
- `배우자 5할 가산`
- `처 감산(남자의 1/2)`
- `비동일가적, 감산(남자의 1/4)`
- `동일가적 여자, 감산(남자의 1/2)`
- `호주상속 5할 가산`
- `1991년 이전 처 사망(대습상속 불가)`
- `재혼(대습상속 불가)`
- `선순위 직계비속 존재`

판정 로직은 사용자의 입력을 바탕으로 작동합니다. 따라서 판정이 이상해 보이면 먼저 데이터 입력 탭에서 관계, 사망일, 혼인·재혼·이혼·복적일, 호주·가적 상태, 제외 사유를 확인해야 합니다.

### 5.4 주요 용어

| 용어 | 의미 |
|---|---|
| 선사망자 | 현재 사건의 피상속인보다 먼저 사망한 사람 |
| 후사망자 | 현재 사건의 피상속인보다 나중에 사망한 사람 |
| 대습상속 | 선사망자 계통의 유효한 하위 상속인이 대신 상속하는 흐름 |
| 재상속 | 후사망자가 받은 지분이 그 사람의 사망으로 다시 상속되는 흐름 |
| 동일가적 | 구법 사건에서 같은 가적에 있는 상태 |
| 비동일가적 | 구법 사건에서 다른 가적에 있는 상태 |
| 그룹통분 | 같은 가족 그룹 안에서 비교하기 위해 맞춘 공통분모 |
| 전체통분 | 모든 최종 상속인의 지분을 비교하기 위해 맞춘 공통분모 |

### 5.5 확인 체크리스트

- 사망일 기준 적용 법 시기가 맞는가
- 선사망자와 후사망자가 현재 사건 기준일에 맞게 구분되는가
- 후사망자는 재상속 사건으로 이어지는가
- 선사망자는 대습상속 또는 제외로 정리되는가
- 구법 사건의 동일가적·비동일가적 입력이 맞는가
- 호주상속 가산 대상자가 맞게 지정되었는가
- 제외 사유가 판정 요소에 맞게 표시되는가

### 5.6 짧은 툴팁 문구

```text
사망일, 관계, 생사 상태, 호주·가적 정보, 제외 사유를 기준으로 각 사건의 상속권과 지분을 판정합니다.
```

## 6. 도움말 UI 배치 기준

각 탭 상단에는 작은 도움말 버튼을 둔다.

권장 버튼 라벨:

```text
?
```

또는 아이콘 라이브러리를 사용할 수 있으면 `CircleHelp` 아이콘을 사용한다.

도움말은 모달보다 우측 또는 중앙 드로어가 적합하다. 사용자가 현재 화면을 보면서 설명을 읽을 수 있어야 한다.

도움말 패널 구성:

1. 탭 제목
2. 한 줄 설명
3. 핵심 본문
4. 확인 체크리스트
5. 관련 용어 또는 번호 설명

긴 법률 설명은 도움말에 넣지 않는다. 필요한 경우 별도 상세 문서나 판정 로직 도움말로 연결한다.
