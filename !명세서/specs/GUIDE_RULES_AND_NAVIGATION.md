# Guide Rules and Navigation

본 문서는 smart guide가 언제 생성되고, 어떤 우선순위로 노출되며, 사용자를 어느 탭과 노드로 이동시키는지 정리한 구현 기준 문서다.

---

## 1. 가이드의 목적

현재 가이드는 단순 설명문이 아니라, 사용자가 다음에 무엇을 해야 하는지 알려 주는 행동 유도 레이어다.

가이드는 주로 `src/hooks/useSmartGuide.js`에서 생성되며:

- 입력 누락을 보완하게 하고
- 구조 오류를 정리하게 하며
- 자동 탐색 전에 확인을 유도하고
- 구법 사건의 추가 검토 포인트를 알려 준다.

---

## 2. 가이드 분류

### 2.1 `mandatory`

계산 결과나 법리 판단에 직접 영향을 주는 필수 확인 사항.

예:

- 피상속인 이름/사망일 미입력
- 사망자로 표시했지만 사망일 없음
- 선사망 자녀/형제자매의 하위 상속인 검토 필요
- 중복 배우자 정리 필요
- 구법 사건의 호주 지정 필요

### 2.2 `recommended`

계산을 막지는 않지만 법리 정확도와 입력 정합성을 높이기 위한 권고 사항.

예:

- 구법 여성 상속인의 혼인·복적 정보 확인
- 호주 검토 대상 사건의 추가 검토
- 독립 사건에서 제외 상태 재확인

---

## 3. 생성 우선순위

현재 구현상 중요한 우선순위는 다음과 같다.

1. 피상속인 기본정보 누락
2. 구조 오류
3. 사망자 기본정보 누락
4. 선사망/후사망 관련 후속 상속인 확인
5. 중복 배우자
6. 구법 호주 검토
7. 구법 여성 상속인 검토
8. 기타 recommended 가이드

특히 루트 이름이나 사망일이 없으면 다른 복잡한 가이드보다 먼저 초기 가이드 하나를 보여 준다.

---

## 4. 핵심 가이드 규칙

### 4.1 초기 필수 정보 가이드

조건:

- `tree.name`이 비어 있거나
- `tree.deathDate`가 없음

동작:

- 다른 복잡한 가이드보다 먼저 노출
- `targetTabId: 'root'`
- 사용자를 기본 입력부터 하게 한다

### 4.2 구조 오류 가이드

조건:

- 비속 아래에 `parent`, `sibling` 같은 관계가 잘못 배치됨

동작:

- `mandatory`
- 필요 시 삭제 행동 유도
- `targetNodeId`, `parentNodeId`를 함께 가질 수 있음

### 4.3 사망일 누락 가이드

조건:

- `isDeceased === true`
- `deathDate` 없음

동작:

- 해당 인물 입력 위치로 이동 유도

### 4.4 선사망 후속 상속인 검토 가이드

조건:

- 피상속인보다 먼저 사망했고
- 자녀/형제자매 관계이며
- 하위 상속인이 비어 있고
- 사용자가 `successorStatus`로 확정하지 않음

동작:

- `mandatory`
- 여러 명이면 그룹화해 표시 가능
- 개별 이동 또는 부모 탭 이동으로 연결됨

### 4.5 직접 후속 상속인 누락 가이드

조건:

- 후사망자 또는 별도 사건 피상속인인데
- 하위 상속인이 비어 있고
- `successorStatus` 확정이 없음

동작:

- 선사망 대습 검토와 구분해서 표시
- 그룹화될 수 있음

### 4.6 중복 배우자 가이드

조건:

- 같은 인물 아래 유효 배우자가 2명 이상 존재

동작:

- `mandatory`
- 실제 상속받는 배우자 1명만 남기고 나머지를 정리하도록 유도

### 4.7 호주 검토 가이드

조건:

- 구법 사건이고
- 1차 상속인이 존재하는데
- 유효 호주 승계인이 지정되지 않음

동작:

- `mandatory` 또는 `recommended`
- `navigationMode: 'event'`를 사용해 사건 단위 검토를 유도할 수 있음

### 4.8 차순위 여성 상속인 검토 가이드

조건:

- 구법 사건
- 후속 상속인 없음 확정 또는 차순위 검토 상황
- 형제자매 중 여성 상속인의 혼인/복적 정보가 불충분

동작:

- `recommended`
- 자매 출가녀 감산이 결과를 바꿀 수 있음을 안내

---

## 5. 가이드 그룹화 규칙

현재 구현은 같은 성격의 가이드를 묶어서 보여 줄 수 있다.

주요 그룹:

- 선사망 후속 상속인 누락 그룹
- 직접 후속 상속인 누락 그룹
- 차순위 여성 상속인 검토 그룹

그룹화 목적은 같은 원인을 여러 줄로 반복하지 않고, 사용자가 한 번에 정리할 수 있게 하기 위함이다.

---

## 6. navigation 필드 의미

### 6.1 `targetTabId`

사용자를 우선적으로 이동시킬 탭 또는 사건의 기준 식별자.

### 6.2 `targetNodeId`

탭 안에서 특정 인물 행이나 노드를 직접 가리킬 때 사용.

### 6.3 `navigationMode`

현재 구현에서는 특히 `event` 모드를 사용해, 인물 자체가 아니라 사건 단위 검토 화면으로 안내할 수 있다.

---

## 7. Guide, Warning, Audit의 차이

문서상 이 셋을 섞어 쓰면 안 된다.

- **Guide**: 사용자가 다음에 할 행동
- **Warning**: 엔진이 계산 중 감지한 경고
- **Audit**: 최종 결과 무결성 검사

예:

- “중복 배우자를 정리해 주세요”는 guide
- `ineligible-substitution-heirs`는 warning
- `final-total-mismatch`는 audit

---

## 8. 현재 구현 기준으로 꼭 반영해야 하는 항목

문서 최신화 시 다음 항목은 반드시 빠지면 안 된다.

- `mandatory` / `recommended` 구분
- `targetTabId`, `targetNodeId`, `navigationMode`
- duplicate spouse 가이드
- missing heirs 그룹화
- hoju review 가이드
- next-order female review 가이드
- import 이후에도 가이드가 다시 생성되는 흐름

---

## 9. 부정확한 문서 표현 예시

다음 표현은 현재 구현과 맞지 않는다.

- “가이드는 계산 결과를 직접 차단한다”
- “경고와 가이드는 같은 것이다”
- “후속 상속인 누락은 모두 하나의 규칙으로 처리된다”
- “중복 배우자는 audit만 처리한다”

현재 구현은:

- 가이드는 행동 유도 레이어이고
- warning/audit와 분리되며
- 누락 유형도 선사망 대습 검토와 직접 후속 상속 검토로 나눠 다룬다.
