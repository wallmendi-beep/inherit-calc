# 스마트 가이드 규칙 및 바로가기 매핑

본 문서는 스마트 가이드가 **언제 뜨는지**, **어떤 문구로 보이는지**, **카드를 클릭하면 어디로 이동하는지**를 한 번에 확인하기 위한 운영 문서입니다.

---

## 1. 가이드 분류 원칙

### 1.1. 필수 검토
- 계산 또는 법리 판단에 직접 영향을 주는 항목
- 사용자가 확인하지 않으면 계산 결과를 오해할 수 있는 항목
- 기본적으로 상단에 노출

### 1.2. 권장 가이드
- 계산은 가능하지만, 법리·입력 정합성을 위해 확인하면 좋은 항목
- 사용자가 의도적으로 유지할 수도 있는 항목
- 체크하여 `확인된 항목`으로 내려보낼 수 있음

### 1.3. 참고사항 / 무결성 경고
- 엔진이 직접 올리는 경고 또는 결과 검산 이슈
- 결과표, 요약표, 금액표와 함께 보는 용도

---

## 2. 가이드 생성 규칙

### 2.1. 초기 필수 가이드
- 조건:
  - 피상속인 성명 누락
  - 피상속인 사망일 누락
- 문구 예시:
  - `사건번호와 피상속인의 기본정보를 입력해주세요.`
- 이동 위치:
  - `root`

### 2.2. 선사망 자녀 + 하위 상속인 없음
- 조건:
  - 선사망
  - 자녀 또는 형제자매
  - 활성 하위 상속인 없음
- 문구 예시:
  - `선사망자입니다. 대습상속인이 있으면 입력하세요.`
- 이동 위치:
  - 해당 사망자의 탭 (`personId`)

### 2.3. 후사망 + 후속 상속인 없음
- 조건:
  - 후사망
  - 활성 하위 상속인 없음
- 문구 예시:
  - `후속 상속인이 없으면 차순위 상속인에게 자동 분배됩니다.`
- 이동 위치:
  - 해당 사망자의 탭 (`personId`)

### 2.4. 선사망 배우자
- 조건:
  - 배우자
  - 피상속인보다 먼저 사망
- 처리 원칙:
  - 선사망 배우자는 본래 상속권이 없으므로 **단독 필수 가이드로 띄우지 않는다.**
  - 배우자 중복 검사에서도 제외한다.
- 이동 위치:
  - 없음

### 2.5. 사망일 미입력
- 조건:
  - `isDeceased = true`
  - `deathDate` 없음
- 문구 예시:
  - `[이름]은 사망자로 표시되어 있으나 사망일이 없습니다.`
- 이동 위치:
  - **부모 탭 (`parentTabId`)**
- 이유:
  - 현재 UI 구조상 사망일은 부모 탭의 입력행에서 수정하는 방식이므로, 해당 탭으로 보내는 것이 맞다.

### 2.6. 혼인일 / 출가 확인
- 조건:
  - 구법 사건
  - 딸
  - 혼인정보 없음
  - 동일가적으로 남아 있음
- 문구 예시:
  - `구법 적용 대상입니다. 혼인일자 또는 출가 설정을 확인하세요.`
- 이동 위치:
  - **부모 탭 (`parentTabId`)**
- 이유:
  - 현재 UI 구조상 혼인일과 출가/동일가적 수정도 부모 탭 입력행에서 수행한다.

### 2.7. 유효 배우자 중복
- 조건:
  - 같은 기준 인물 아래 유효 배우자 2명 이상
  - 단, 선사망 배우자 / 제외 배우자는 제외
- 문구 예시:
  - `유효 배우자가 2명 이상입니다. 1명만 남기세요.`
- 이동 위치:
  - 해당 기준 인물 탭 (`personId`)

### 2.8. 구법 호주 지정 누락
- 조건:
  - 구법 사건
  - 호주 지정 대상
  - 하위 상속인 있음
  - `isHoju` 지정 없음
- 문구 예시:
  - `구법 적용 대상입니다. 호주상속인을 지정하세요.`
- 이동 위치:
  - 해당 피상속인 탭 (`personId`)

### 2.9. 연쇄 호주 승계 팁
- 조건:
  - 구법 사건
  - 현재 인물이 호주
  - 사망자
  - 하위에 새 호주 지정 없음
- 문구 예시:
  - `연쇄 호주 승계라면 두 사람 모두 호주상속 켬 상태를 확인하세요.`
- 이동 위치:
  - 해당 피상속인 탭 (`personId`)

### 2.10. 독립 제외(상속포기/상속결격)
- 조건:
  - `isExcluded = true`
  - `exclusionOption in ['renounce', 'disqualified']`
  - 단, 선사망 파생 제외는 제외
- 문구 예시:
  - `[이름] 상속포기가 적용되었습니다. 타 피상속인 탭도 확인하세요.`
- 이동 위치:
  - 없음 또는 단순 안내

### 2.11. 감사 엔진 무결성 이슈
- 조건 예시:
  - 최종합 불일치
  - 미전달 지분
  - 사망자 지분 최종 결과 잔존
  - 계층 위반
- 이동 위치 원칙:
  - 가능한 경우 `issue.targetTabId`
  - 없으면 `personId`
  - 둘 다 없으면 `root`

---

## 3. 문구 단순화 규칙

- 카드 본문은 가능하면 **1문장**
- 권장 길이: **40자 내외**
- 구조:
  - `현상 + 조치`
  - 예: `유효 배우자가 2명 이상입니다. 1명만 남기세요.`
- 장문 설명은 피하고, 클릭 이동으로 보완한다.

---

## 4. 카드 클릭 시 이동 규칙

### 4.1. 기본 원칙
- 클릭 시 `입력 탭`으로 이동한다.
- 해당 인물이 속한 피상속인 탭을 연다.
- 해당 행을 스크롤/하이라이트한다.

### 4.2. 예외
- 사망일 미입력: 부모 탭으로 이동
- 혼인일/출가 확인: 부모 탭으로 이동
- 전역 무결성 경고: 가능한 한 원인 인물 탭으로 이동, 불가하면 root

---

## 5. 확인 체크(의도적 유지) 규칙

### 5.1. 의미
- 체크는 `오류 해결`이 아니라,
  **사용자가 의도적으로 확인했고 당장은 유지하기로 한 상태**를 뜻한다.

### 5.2. 동작
- 체크한 가이드는 기본 목록에서 내려간다.
- 하단의 `확인된 항목` 섹션으로 이동한다.
- 섹션은 접기/펼치기 가능한 아코디언으로 둔다.

### 5.3. 자동 제거
- 해당 가이드의 원인이 실제로 해소되면,
  `확인된 항목` 목록에서도 자동 제거한다.
- 즉, 체크 상태는 영구 보존이 아니라 **현재 살아 있는 같은 가이드에만 연결**된다.

### 5.4. 권장 적용 대상
- 권장 가이드
- 반복 확인이 필요한 선사망 자녀/대습상속 안내
- 사용자가 의도적으로 미입력 상태를 유지하는 경우

---

## 6. 디자인 규칙

- 상태 이모지/아이콘은 **섹션 헤더에만 1회 사용**
- 카드별 반복 아이콘은 지양
- 카드 내부는 문구와 배경/테두리 톤 차이로 위계를 만든다
