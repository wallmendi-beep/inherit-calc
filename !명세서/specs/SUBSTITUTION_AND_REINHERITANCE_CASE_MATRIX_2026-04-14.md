# 대습상속 / 재상속 경우의 수 정리

본 문서는 `데이터 입력 = vault`, `시뮬레이션 = 사건별 상속 흐름`, `계산 상세 = 법리/수치 검산` 원칙 아래에서  
대습상속과 재상속의 경우의 수, 가이드 생성 조건, 입력 확정 흐름을 정리한 SSOT이다.

## 1. 기본 원칙

- `대습상속`은 **선사망자 기준**으로 열린다.
- `재상속`은 **후사망자 기준**으로 열린다.
- 입력 탭은 사람과 사실 상태를 입력하는 곳이다.
- 사건별 법리 결과는 시뮬레이션과 계산 상세에서 보여준다.
- 후속 상속인이 실제로 없으면 입력 탭에서 `없음 확정`으로 마감할 수 있다.

## 2. 대습상속

### 2-1. 정의

- 원래 상속인이 될 사람이 피상속인보다 먼저 사망한 경우
- 그 배우자 또는 직계비속이 대신 상속에 들어오는 구조

### 2-2. 입력 흐름

1. 부모 단계 행에서 선사망 상속인을 본다.
2. 아래 둘 중 하나를 선택한다.
   - `대습상속 >>`
   - `대습상속인 없음`
3. `대습상속인 없음`을 누르면 그 행은 입력상 마감 상태가 된다.
4. `대습상속 >>`를 누르면 그 사람의 후속 상속인 입력 단계로 들어간다.

### 2-3. 경우의 수

#### A. 선사망자에게 배우자/직계비속이 있음

- 부모 행에서 `대습상속 >>` 사용
- 하위 상속인을 입력하거나 `불러오기`
- 구법이면 그 하위 사건에서 `호주상속 / 재산상속` 판단 가이드가 열릴 수 있음

#### B. 선사망자에게 배우자/직계비속이 없음

- 부모 행에서 `대습상속인 없음` 확정
- 같은 이유의 반복 가이드는 더 이상 띄우지 않음
- 이후 사건은 차순위 상속 검토로 넘어갈 수 있음

#### C. 선사망자가 현재 사건에서 대습불가인 경우

예:
- 1991년 이전 사위 대습불가

원칙:
- 현재 사건에서는 배제될 수 있음
- 그러나 그 사람 자신의 사건 입력 가능 여부는 별도로 본다
- 즉 `현재 사건에서의 상속권 배제`와 `자기 사건 입력 가능 여부`를 분리한다

### 2-4. 대습상속에서 호주 가이드가 필요한 경우

아래를 모두 만족할 때:

- 구법(1991년 이전) 사건
- 실제로 대습상속인이 입력됨
- 그 대습상속인의 내부 분배에서 `호주상속 / 재산상속` 구분이 결과를 바꿀 수 있음

주의:
- 단순히 선사망이라는 이유만으로 무조건 호주 가이드를 띄우지 않는다
- 현재 사건에서 법리상 바로 배제되는 사람에게는 호주 가이드를 띄우지 않는다

## 3. 재상속

### 3-1. 정의

- 현재 사건에서 실제 지분을 받은 사람이
- 나중에 다시 사망하면서
- 그 지분이 자신의 상속인에게 넘어가는 구조

### 3-2. 입력 흐름

1. 후사망 상속인을 본다.
2. 아래 둘 중 하나를 선택하거나 사용한다.
   - `재상속 >>`
   - `직계비속·배우자 없음`
   - `추가 상속인 없음` (배우자 계열)
3. 후속 상속인이 있으면 `재상속 >>`로 들어가 입력한다.
4. 후속 상속인이 실제로 없으면 `없음 확정`으로 마감한다.

### 3-3. 경우의 수

#### A. 후사망자에게 배우자/직계비속이 있음

- `재상속 >>`
- 하위 상속인 입력 또는 `불러오기`
- 구법이면 그 사건에서 다시 `호주상속 / 재산상속` 판단 가이드 가능

#### B. 후사망자에게 배우자/직계비속이 없음

- `직계비속·배우자 없음` 또는 `추가 상속인 없음` 확정
- 반복 가이드 중단
- 차순위 상속으로 진행

#### C. 피상속인의 남편/처가 후사망한 경우

원칙:
- 현재 사건에서는 정상 상속 가능
- 그 사람 자신의 사건은 재상속 사건으로 따로 입력/검토
- 구법이면 그 사건 안에서 다시 호주상속 여부 판단 가능

### 3-4. 재상속에서 호주 가이드가 필요한 경우

아래를 모두 만족할 때:

- 구법(1991년 이전) 사건
- 그 사람이 현재 사건 지분을 실제로 받음
- 후속 상속인이 입력됨
- 그 후속 상속 사건에서 `호주상속 / 재산상속` 구분이 결과를 바꿈

## 4. 차순위 상속으로 넘어가는 경우

### 4-1. 기본 원칙

- `대습상속인 없음`
- `직계비속·배우자 없음`
- `추가 상속인 없음`

중 하나로 마감하면, 그 사건은 차순위 상속 검토로 넘어갈 수 있다.

### 4-2. 여성 차순위 상속인 검토

구법에서 차순위 상속인 중 여성이 있으면:

- 이미 `vault`에 혼인/이혼/복적 정보가 있는 사람은 다시 묻지 않는다
- 정보가 비어 있는 사람만 안내한다
- 평생 미혼이 확실하면 `동일가적`으로 바로 확정 가능

가이드 문구 원칙:

- 해당 사망 사건을 기준으로 묶어서 안내
- 사람별로 같은 문구를 여러 번 반복하지 않음

예:

- `[정문자] 사건은 차순위 상속으로 진행됩니다. 여성 형제자매 중 동일가적 여부 확인이 필요한 사람이 있습니다: [...]`

## 5. 가이드 생성 원칙

### 5-1. 입력 탭

- 사실 입력용 가이드만 표시
- 다음 중 하나를 요구:
  - 후속 상속인 입력
  - 없음 확정
  - 구법 여성의 혼인/복적 정보 확인
  - 구법 호주상속 판단을 위한 후속 상속인 확인

### 5-2. 시뮬레이션 / 계산 상세

- 사건별 법리 결과와 경고 표시
- 왜 지분이 그렇게 이동했는지 설명
- 이상한 사람 이름을 누르면 입력 탭으로 이동해 수정 가능

### 5-3. 반복 금지

- 같은 성격의 가이드는 개별 카드 반복 대신 목록형으로 묶는다
- `없음 확정`된 사람에게 동일 사유의 경고를 다시 띄우지 않는다

## 6. 실무용 한 줄 규칙

- 대습상속 = 선사망자 기준
- 재상속 = 후사망자 기준
- 후속 상속인이 있으면 입력
- 없으면 `없음 확정`
- 구법에서 결과가 달라질 수 있으면 `호주상속 / 재산상속` 확인
- 차순위 상속으로 넘어가면 여성 상속인의 `동일가적 / 비동일가적` 사실 입력을 다시 점검
