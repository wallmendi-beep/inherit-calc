# ⚖️ 상속지분 계산기 PRO - 핵심 로직 및 가이드 명세 (CORE_SPECS)

본 문서는 상속지분 계산기의 법정 상속 로직, 계보 추적 엔진, 그리고 스마트 가이드 안내 문구의 표준을 정의하는 **최종 기능 명세서**입니다.

---

## 1. 명칭 및 호칭 표준 (Naming)

### 1.1. 상속인 관계 및 호칭
*   **자녀**: '아들', '딸' (1991년 이전 법률 적용 대상 구분 필수)
*   **배우자**: '처', '남편' (민법 개정 시기에 따른 명칭 자동 전환)
*   **존칭 배제**: 모든 시스템 UI 및 안내 문구에서 **"님" 존칭을 일괄 제거**합니다. (예: `홍길동님` → `홍길동`)

### 1.2. 계보상 위치 명명
*   **최상위**: '피상속인' (상속 개시의 중심)
*   **하위 탭**: '피대습상속인' (선사망자) 또는 '피상속인' (후사망/재상속자)으로 상황에 맞게 명시합니다.

---

## 2. 사망 시차별 상속 전이 및 토글 규칙 (v1.7)

상속인의 사망 시점(선사망/후사망)에 따라 시스템은 서로 다른 기본값과 조작 권한을 부여합니다.

### 2.1. 선사망 (대습상속)
*   **상황**: `상속인 사망일 < 피상속인 사망일`
*   **기본 상태**: [상속에서 제외(Off)]
*   **자동 복구 규칙**: 선사망 자녀/형제자매에게 하위 상속인(대습상속인)이 1명 이상 입력되면, 시스템은 이를 `상속권 없음`이 아니라 **`대습상속 진행` 상태**로 재판정하고 상태 토글을 기본 ON으로 복구합니다.
*   **배우자 예외**: 선사망 배우자는 대습상속 자동 복구 대상이 아니며, 기본적으로 OFF 상태를 유지합니다.
*   **가드레일**: 사용자가 토글을 ON으로 변경하면 **3초간 시각적 ON 상태**를 유지합니다. 하위에 상속인 데이터(대습상속인)가 추가되지 않으면 3초 후 자동으로 다시 OFF 처리됩니다.
*   **지분 표시**: 대습상속인 입력 전까지는 `0`으로 표시됩니다.

### 2.2. 후사망 (재상속)
*   **상황**: `상속인 사망일 >= 피상속인 사망일`
*   **기본 상태**: **[상속인 포함(On)]**
*   **지능형 전환**: 사망일 입력 시 후사망으로 판별되면 즉시 제외 상태를 해제(`isExcluded: false`)하고 본래의 상속지분(1분의 1 등)을 노출합니다.

---

## 3. 스마트 가이드 및 자동 분배 추정 (Guidance)

### 3.1. 초기 단계 가이드 (제7조)
*   피상속인의 **[성명]** 또는 **[사망일자]**가 누락된 경우, 타 가이드보다 **"사건번호와 피상속인의 기본정보를 입력해주세요."**를 최우선 단독 노출합니다. 기초 정보 누락 시 복잡한 추정 로직 노출을 억제합니다.

### 3.2. 재상속 예측 및 실명 안내
*   하위 상속인이 미입력된 후사망자 탭 진입 시, 가계도를 역추적하여 상속 권리 전이 경로를 실명으로 고지합니다.
*   **가이드 예시**: "별도의 상속인을 입력하지 않으면 [피상속인]의 직계비속인 [실명1], [실명2]에게 상속지분이 자동으로 분배됩니다."

---

## 4. Vault 기반 다계층 상태 관리

*   **독립적 상태**: 한 인물이 다중 상속 관계를 맺을 때, 각 탭의 피상속인 사망 시점과의 대조를 통해 '출가녀/동일가적/호주' 여부를 독립적으로 산출합니다.
*   **날짜 메모리**: 날짜 입력 후 삭제하더라도 내부적으로 백업(`_lastDate`)하며, 비어있는 입력란 옆에 **[복원]** 버튼을 노출합니다.
*   **상실선고 필터**: 2024년 4월 25일 이후 상속 개시 사건에서만 해당 옵션을 노출합니다.

## 6. 실시간 조건 추적 및 자동 동기화 원칙 (v4.33)

### 6.1. 기본 원칙
* 모든 파생 상태는 **현재 입력값 전체**를 기준으로 실시간 재판정한다.
* 사용자의 마지막 클릭 결과를 임시 상태로 고정하지 않는다.
* 입력 필드와 판정 로직은 분리한다.
  * 예: 체크박스는 생존/사망 상태만 담당
  * 예: 날짜 입력은 사망일자 값만 담당
* 상위 노드가 바뀌면 하위 노드의 판정, 가이드, 계산 결과도 즉시 다시 계산되어야 한다.

### 6.2. 자동 동기화 규칙
* 부모 또는 배우자의 사망일이 뒤늦게 입력되거나 수정되면, 그 영향을 받는 자녀, 대습상속인, 재상속인의 상태를 즉시 재판정한다.
* 선사망/후사망 판정은 저장된 토글 상태가 아니라, `사망일자`와 `기준 사망일자`의 비교로 매번 다시 계산한다.
* 선사망 자녀/형제자매에게 하위 상속인이 새로 입력되면, 기존 `predeceased` 제외 상태를 자동 해제하고 `대습상속 진행` 상태로 전환한다.
* 반대로 선사망자에게 하위 상속인이 하나도 없으면, `상속권 없음 (선사망)` 상태를 유지한다.
* 상속권 포함/제외, 대습상속, 재상속, 자동 분배 안내 문구는 입력 순서와 무관하게 동일한 최종 조건을 반영해야 한다.
* 사용자가 일부 정보를 나중에 입력하더라도, 기존 계산 결과에 머무르지 않고 최신 조건으로 전체 동기화한다.

### 6.3. 날짜 메모리 복원 규칙
* 사망자를 생존으로 전환하면 화면의 사망일자는 비우되, 마지막 입력값은 `_lastDeathDate`에 보관한다.
* 다시 사망으로 전환하면 `_lastDeathDate`를 자동 복원한다.
* 이 메모리 값은 Vault 저장/복원 과정에서도 유지되어야 한다.
* 날짜를 지우거나 다시 입력하는 행위가 생존/사망 토글 상태를 임의로 변경해서는 안 된다.

### 6.4. 구현 원칙
* UI는 파생 상태를 직접 확정하지 않고, 엔진/도메인 로직이 계산한 결과를 표시한다.
* 특정 입력 변경이 다른 노드의 법적 지위에 영향을 주는 경우, 해당 가지(branch)뿐 아니라 필요한 상위/하위 노드 전체에 재평가를 전파한다.
* 계산 결과, 가이드, 배지, 버튼 문구는 동일한 판정 소스를 공유해야 한다.
