# 탭 역할 정의 및 사건 검토 뷰 설계

작성일: 2026-04-13  
최종 업데이트: 2026-05-04 (사건 검토 마인드맵 및 보고서 사이드패널 설계 반영)  
적용 범위: `데이터 입력`, `사건 검토`, `계산 상세`, `지분 요약`, `구체적 상속분`

---

## 1. 목적

본 문서는 상속지분 계산기 각 탭의 역할을 정의하고,  
특히 `사건 검토` 탭의 설계 철학과 구현 원칙을 기록한다.

핵심 목표:

1. `데이터 입력` 탭은 사실관계 입력에만 집중한다.
2. `사건 검토` 탭은 **피상속인별 상속 흐름을 독립 사건 단위로 시각화**하여 입력 오류를 검증한다.
3. `계산 상세` 탭은 법리와 수치 계산의 테이블 검산 화면이 된다.
4. `지분 요약` 탭은 최종 지분을 두 가지 시각(구조별/경로별)으로 제공한다.
5. 사용자는 어느 탭에서든 사람 이름을 눌러 입력 탭으로 이동할 수 있어야 한다.

## 3. 탭별 역할 SSOT

### 3.1 데이터 입력

한 줄 원칙: **`사실관계 원본 구축`**

- vault 입력/수정
- 사람, 관계, 날짜, 상태, 호주/가적 조건 입력
- 사이드바(가계도 요약) 기본 열림 상태

### 3.2 사건 검토 (구: 가계도/시뮬레이션)

한 줄 원칙: **`피상속인별 상속지분 분기와 판정 근거 독립 검증`**

#### 설계 철학

이 탭의 가장 중요한 설계 원칙은 **인지 부하 분산**과 **역할 분리**이다.

전체 상속 연쇄를 한 화면에서 보려 하면 인간의 인지 한계를 초과한다.  
따라서 `한 사망 사건 = 독립된 검토 단위`로 쪼개어, 각각을 별도 사건으로 확인한다.

또한 가계도, 분기 그래프, 보고서의 역할을 섞지 않는다.

- 가계도: 누가 누구의 가족인지 확인한다.
- 상속지분 마인드맵: 지분이 어디로 분기되는지 확인한다.
- 사건 보고서: 선택한 사건에서 왜 그런 판정이 나왔는지 확인한다.

이 탭이 해결하는 세 가지 문제:

1. **시점별 조건 변화**: 동일 인물이라도 피상속인의 사망 시점에 따라 관계/조건이 달라진다 (예: 혼인 전후, 출가녀 여부, 1991년 이전후)
2. **상속지분 분기 검증**: 복수 경로·재상속·대습상속에서 지분이 어디로 갈라지는지 시각적으로 확인
3. **제외 이유 명시**: 상속권이 없는 인물과 그 이유(선사망, 혼인/재혼, 1991년 이전 제한 등)를 사건 보고서에 명시

#### 이 탭의 전신

원래 이 탭은 순수 가계도 확인 탭이었다.  
그러나 가계도 역할은 `사이드바(가계도 요약)`가 담당하게 되어,  
이 탭은 `피상속인 중심의 데이터 입력 검증 탭`으로 재설계되었다.

#### 화면 구조

```
[좌측 사건 보고서 사이드패널]            |   [우측 상속지분 마인드맵]
                                         |
  사건 보고서                            |   김혁조
  피상속인 김명수  상속지분 1분의 1       |     ├ 구수명 2/15
  사망일자 1967년 10월 27일  적용법 구민법 |     │   └ 재상속 분기
                                         |     ├ 김명남 1/15
  사건 관계인                            |     └ 김명수 1/15
  성명 / 관계 / 지분 / 가감산 요소         |
                                         |   선택 노드가 포함된 계통을
  사건 요약                              |   확대·접기·펼치기로 검토
  상속권 없음                            |
  가감산 요소                            |
  재상속·대습상속                        |
```

#### 사이드패널 역할

`데이터 입력` 탭에서는 좌측 사이드패널이 상속가계도 역할을 한다.  
`사건 검토` 탭에서는 같은 위치의 사이드패널이 사건 보고서 역할을 한다.

이 구조는 다음을 목표로 한다.

- 마인드맵 영역을 넓게 확보한다.
- 보고서는 좁은 세로 문서로 유지한다.
- 노드 카드에는 최소 정보만 두고, 판정 근거는 보고서에서만 설명한다.
- 상속권 없는 사람은 그래프 노드에서 제외하되, 사건 보고서에는 빠짐없이 표시한다.

#### 사건 보고서 구성

사건 보고서는 선택한 노드가 속한 **하나의 상속사건**만 다룬다.  
원자료 전체 설명이나 취득경로 요약은 넣지 않는다.

상단에는 `사건 보고서` 제목을 두고, 기본정보는 두 줄로 압축한다.

```
사건 보고서
피상속인 김명수   상속지분 1분의 1
사망일자 1967년 10월 27일   적용법 구민법
```

적용법 표기는 다음 3개로 통일한다.

- `구민법`
- `79년 민법`
- `현행민법`

본문 섹션은 다음 3개만 둔다.

1. `사건 관계인`
   - 컬럼: `성명 / 관계 / 지분 / 가감산 요소`
   - 사건 관계인 표의 지분은 `2/165`처럼 분수로 표시한다.
   - `판정` 컬럼은 두지 않는다.
   - `상속권 없음`은 가감산 요소 칸에 표시한다.

2. `사건 요약`
   - `상속권 없음`
   - `가감산 요소`
   - `재상속·대습상속`

3. `판정 요약`
   - 표와 사건 요약만으로 부족한 경우에만 짧게 보충한다.

#### 사건 요약 문구 원칙

데이터 입력에서 수집한 날짜와 상태값을 최대한 활용한다.  
다만 문장은 길게 쓰지 않고 `[이름] + 날짜/사유 + 결과` 형식으로 통일한다.

```
상속권 없음
- [정문자] 1962년 05월 10일 혼인으로 상속권 없음
- [정문자] 1968년 02월 03일 재혼으로 상속권 없음

가감산 요소
- [정옥재] 1965년 03월 12일 출가로 감산
- [정인숙] 출가로 감산
- [김철수] 호주상속으로 가산

재상속·대습상속
- [구수명] 1979년 00월 00일 사망으로 재상속
- [정문자] 0000년 00월 00일 사망으로 대습상속
```

날짜가 없는 경우에는 날짜를 생략한다. `날짜 미상` 같은 문구를 반복하지 않는다.

### 3.3 계산 상세

한 줄 원칙: **`법리와 수치 검산`**

- 사건별 계산식 테이블: `이름 / 관계 / 계산식 / 계산결과 / 비고`
- 호주가산 안내 카드
- 인쇄용 포맷

### 3.4 지분 요약 (구: 법정 상속분 + 결과 리포트)

한 줄 원칙: **`최종 결과 두 가지 시각`**

두 개의 뷰 모드를 내부 토글로 제공:

| 뷰 | 구 탭 | 설명 |
|----|-------|------|
| 지분 구조 | 법정 상속분 | 가계도 구조 기준 / 대습·재상속 그룹 / 통분 검증 |
| 취득 경로 | 결과 리포트 | 상속인별 / 어떤 사건에서 얼마 받았는지 역추적 |

통합 근거:
- 두 뷰 모두 `finalShares + calcSteps` 같은 데이터를 다른 정렬 방식으로 표현
- 6개 탭에서 5개 탭으로 축소하여 인지 부하 감소

### 3.5 구체적 상속분

한 줄 원칙: **`금액 환산`**

- 상속재산가액 입력
- 특별수익/기여분 입력
- 간주상속재산 및 최종 금액 계산

## 4. 사건 범위 규칙

### 4.1 핵심 원칙

1. `한 사건 = 한 피상속인의 사망` 단위
2. 현재 사건에서는 **1차 상속 흐름**만 보여준다
3. **재상속/대습상속**은 현재 사건에서 펼치지 않고 별도 사건 버튼으로 안내
4. 복수 유입(mergeSources)은 네비게이터에서 참조 관계로만 표시

### 4.2 재상속 절단 원칙

예: `구수명 사망` 사건을 보는 중
- 구수명에서 김명남에게 분배되는 1차 흐름까지만 보여준다
- 김명남이 이후 사망하는 경우는 `김명남 사건`으로 별도 안내
- → `재상속 사건으로 →` 버튼만 표시

### 4.3 인지 부하 분산 근거

김혁조 사건처럼 복잡한 케이스:
- 사망 연도 차이 23년 (1967 vs 1990)
- 같은 인물이라도 사건마다 법적 조건 다름
- 대습상속 + 재상속 복합 구조

전체를 한 화면에서 보려 하면 인간이 이해하기 어려우므로,
사건별로 분리해서 독립 검증하는 것이 더 효과적이다.

## 5. 가계도 사이드바와의 역할 분리

| 기능 | 사이드바(가계도 요약) | 사건 검토 탭 |
|------|----------------------|-------------|
| 가족 구조 파악 | ✓ | - |
| 상속인 누락 확인 | ✓ (input탭 열림시) | - |
| 상속 방향 검증 | - | ✓ |
| 시점별 조건 차이 확인 | - | ✓ |
| 제외 이유 명시 | - | ✓ |

사이드바는 `input탭`에서 기본 열림, `사건 검토 탭` 진입 시 닫힘.

## 6. 미구현 / 향후 과제

1. **사건 네비게이터 들여쓰기 선 연결**: 현재 depth 들여쓰기만 있음, CSS line 연결은 미구현
2. **사이드바 자동 닫힘**: 현재 App.jsx에서 tree탭 진입 시 사이드바 닫힘 처리 확인 필요
3. **시점별 조건 diff 강조**: 동일 인물이 다른 사건에서 다른 조건으로 등장할 때 시각적 차이 표시 (후속 작업)
4. **화살표/연결선**: 사건 간 연결을 시각적으로 강조하는 그래프 뷰 (후속 작업)
