# AI 자동입력 사용자 흐름 및 도움말 기준

최종 수정: 2026-05-07

## 1. 목적

이 문서는 앱의 `AI 자동입력` 기능을 사용자에게 어떻게 설명할지 정리한 기준 문서다.

기존 AI 관련 명세는 주로 JSON 신뢰성, 정규화, 스마트 가이드 연결 원칙을 다뤘다. 그러나 사용자가 실제로 따라야 할 입력 절차, 즉 `전용 프롬프트 복사 -> 외부 AI 사용 -> JSON 붙여넣기 -> 사용자 검수` 흐름은 별도 명세서에 명확히 들어가 있지 않았다.

앱 안의 도움말 모달과 사용자 설명 문서는 이 문서를 기준으로 작성한다.

## 2. 핵심 원칙

`AI 자동입력`은 계산기가 직접 사진이나 PDF를 분석하는 기능이 아니다.

계산기에서 제공하는 전용 프롬프트를 복사한 뒤, 사용자가 사용하는 외부 AI 도구에 자료와 함께 넣어 이 계산기가 읽을 수 있는 JSON을 만들고, 그 JSON을 다시 계산기에 붙여넣는 방식이다.

따라서 AI 자동입력은 자동 계산 기능이 아니라 `입력 초안 생성 + 사용자 검토` 기능이다.

## 3. 사용자 사용 흐름

1. `데이터 입력` 탭에서 `AI 자동입력` 버튼을 누른다.
2. 계산기가 제공하는 전용 프롬프트를 클립보드에 복사한다.
3. 사진, PDF, 판결문, 가족관계 설명, 메모 등 분석할 자료를 사용자가 사용하는 외부 AI 도구에 넣는다.
4. 복사한 전용 프롬프트를 같은 대화에 함께 입력한다.
5. 외부 AI가 이 계산기에서 사용할 수 있는 JSON 형식으로 가족관계 자료를 정리한다.
6. 생성된 JSON 전체를 복사한다.
7. 계산기의 `AI 자동입력` 창에 JSON을 붙여넣고 불러온다.
8. 입력된 상속인 구조, 관계, 날짜, 제외 사유를 사용자가 직접 검토하고 수정한다.

## 4. 입력에 적합한 자료

- 가족관계등록부 내용을 정리한 메모
- 판결문 또는 결정문 중 가족관계와 사망일자가 적힌 부분
- 상속관계 설명 문서
- 상속인 명단
- 사진 또는 PDF 자료를 외부 AI가 읽을 수 있는 형태로 첨부한 것

## 5. 반드시 확인할 항목

AI가 만든 JSON은 입력 초안일 뿐이다. 계산 결과로 사용하기 전 다음 항목을 반드시 확인해야 한다.

- 피상속인과 상속인의 이름
- 배우자, 자녀, 부모, 형제자매 관계
- 사망일자
- 혼인일, 이혼일, 재혼일, 복적일
- 1991년 이전 사건의 호주상속인 여부
- 동일가적 또는 비동일가적 여부
- 선사망, 후사망, 대습상속, 재상속 흐름
- 누락된 상속인 또는 잘못 추가된 인물

## 6. 주의사항

- AI는 자료를 잘못 읽거나 가족관계를 잘못 해석할 수 있다.
- 부모나 형제자매가 자녀 아래에 들어가는 등 구조 오류가 생길 수 있다.
- 1991년 이전 구법 사건의 호주상속, 출가녀 감산, 사위 대습 제한은 반드시 직접 검토해야 한다.
- 원문에는 존재하지만 특정 사망 사건의 상속인이 아닌 후혼 배우자나 후혼 자녀가 함께 들어올 수 있다.
- 개인정보가 포함된 자료를 외부 AI에 넣기 전 주민등록번호, 주소, 연락처 등 불필요한 개인정보는 제거하는 것이 좋다.
- AI 입력 후에는 스마트 가이드와 사건 검토 탭에서 결과를 반드시 검수해야 한다.
