# 배우자 재상속 유저 가이드 수정 계획

작성일: 2026-05-01

## 1. 배경

배우자 재상속 사건의 입력 UX는 과거 `불러오기`를 사용자가 직접 누르는 흐름에서, 배우자 탭 진입 시 상위 사건 자녀를 자동으로 불러오는 흐름으로 바뀌었다. 따라서 유저 가이드는 더 이상 "불러오기부터 하라"는 식으로 안내하지 않고, 자동으로 불러온 목록을 이번 배우자 사건의 상속인 범위에 맞게 검토하도록 안내해야 한다.

## 2. 확정 원칙

1. 배우자 재상속 사건은 처/남편 및 1991년 전후를 불문하고 상위 사건 자녀 목록을 먼저 자동으로 불러온다.
2. 안내 문구는 먼저 자동 불러오기 사실을 알리고, 그 다음 이번 사건에서 할 일을 설명한다.
3. 처 사건은 1991년 이전과 이후를 구분한다.
   - 1991년 이전: 남편의 자녀가 처 사건의 상속인으로 반영될 수 있으므로, 별도 자녀 추가와 제외 여부를 확인한다.
   - 1991년 이후: 배우자의 자녀라는 이유만으로 상속인이 되지 않으므로, 직접 자녀 또는 양자가 아닌 사람은 제외한다.
4. 남편 사건은 처 사건에서 이어진 자녀 목록을 비교 기준으로 보고, 남편의 직접 자녀 또는 양자가 아닌 사람을 제외한다.
5. `추가 상속인 없음`은 자동으로 불러온 사람 외에 더 추가할 상속인이 없다는 확인값이다.
6. 계모자 관계 확인 가이드는 항상 띄우지 않는다. 1991년 이전 처 사건에서 남편 자녀가 처 사건에 빠져 있을 때만 보정 가이드로 표시한다.

## 3. 코드 수정 계획

### 3.1 InputPanel 안내 문구 정리
- 대상: `src/components/InputPanel.jsx`
- 빈 배우자 탭 안내:
  - "아직 직접 입력된 후속 상속인이 없습니다." 대신 자동 불러오기 기준을 설명한다.
  - 자녀가 불러와지면 목록 확인, 없으면 직접 추가 가능하다는 문구로 정리한다.
- 자동 불러오기 배너:
  - 공통 1행: 상위 사건 자녀를 자동으로 불러왔다는 사실
  - 공통 2행: 이 목록은 앞선 사건 기준 자녀이므로 이번 사건의 상속인 범위를 확인하라는 설명
  - 관계/시대별 보충 문구 표시
  - `추가 상속인 없음` 버튼 보조 설명 추가

### 3.2 PersonEditModal 안내 문구 정리
- 대상: `src/components/PersonEditModal.jsx`
- 사건 검토 모달에서도 배우자 재상속 사건을 동일한 원칙으로 설명한다.
- "이 처", "이 남편" 같은 일반 표현 대신 실제 인물명을 사용한다.
- 처 사건은 1991년 이전/이후를 나누어 안내한다.

### 3.3 계모자 가이드 중복 기준 유지
- 대상: `src/hooks/smartGuideHelpers.js`
- 현재 `collectLegacyStepchildGuideEntries`는 1991년 이전이고 남편 자녀가 처 사건 목록에 빠진 경우에만 가이드를 만든다.
- 이번 수정에서는 이 기준을 유지하고, 자동 반영된 자녀에 대해 중복 가이드를 만들지 않는 원칙을 문서화한다.

## 4. 검증 항목

- 배우자 탭 진입 시 자동 불러오기 배너가 시대와 관계에 맞게 표시되는지 확인한다.
- 1991년 이후 처 사건에서 계모자 포함 가능 문구가 노출되지 않는지 확인한다.
- 남편 사건에서 직접 자녀 또는 양자가 아닌 사람을 제외하라는 안내가 표시되는지 확인한다.
- 빌드가 통과하는지 확인한다.
