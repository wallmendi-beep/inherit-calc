# 불러오기 재링크 우선 규칙 (2026-04-13)

## 배경
- `상속인 불러오기`가 대형 사건에서 메모리를 폭증시키는 원인으로 확인되었다.
- 원인은 기존 인물을 다시 불러올 때도 `personId`를 재사용하지 않고, 하위 서브트리를 새 person으로 통째로 복제하던 구조였다.

## 기본 원칙
1. 불러오기 대상이 이미 `vault.persons`에 존재하는 `personId`를 가지면 **기존 person을 재사용**한다.
2. 기존 person을 재사용하는 경우 **하위 서브트리를 다시 복제하지 않는다**.
3. 수동 입력 등으로 아직 `personId`가 없는 새 인물만 새 person을 생성한다.

## 기대 효과
- 같은 가계를 여러 탭에서 다시 불러와도 메모리 폭증 방지
- 동일인 정합성 유지
- 대형 사건에서 `구수명 > 불러오기` 같은 동작 시 수 GB 단위 메모리 증가 방지

## 구현 기준
- `appendResolvedHeirs()`
  - 기존 personId 존재 시 `ensureLink()`만 수행
  - 신규 인물일 때만 person 생성 + 하위 재귀 생성

## 주의
- 같은 부모 아래 동일 targetId 중복 링크는 생성하지 않는다.
- 이 규칙은 `불러오기` 경로에 우선 적용한다.
