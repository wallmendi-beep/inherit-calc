# 입력 탭 특수조건 최소 구성 규칙

작성일: 2026-04-14

## 목적
- 데이터 입력 탭은 사실 입력과 상태 확정에 집중한다.
- 사건별 법리 해석 결과는 `시뮬레이션`, `계산 상세`에서 확인한다.

## 특수조건 칸에 남기는 요소
- 관계 표식
  - 예: `처`, `남편`, `아들`, `딸`
- `호주상속 / 재산상속` 토글
- `동일가적 / 비동일가적` 토글
- `선사망` 표식
- `대습상속인 없음` 확정 버튼
- 배우자/딸의 추가 인적사항 입력 버튼

## 특수조건 칸에서 제거하는 요소
- 가산/감산 배지
- `x 1/2`, `x 1/4`, `x 1.5` 같은 계산 계수
- `사위(대습불가)` 같은 현재 사건 해석 결과 배지
- 사건별 상속권 인정/배제 결론

## 역할 분리 원칙
- 데이터 입력
  - 사람, 관계, 날짜, 선택 상태 입력
- 시뮬레이션
  - 사건별 상속 흐름과 사건 한정 해석 결과 확인
- 계산 상세
  - 계산식, 비고, 가감산, 상속분 검산

## 선사망자 처리
- 선사망자는 특수조건 칸에 `선사망` 표식 표시
- 하위 상속인이 없으면 `대습상속인 없음` 버튼으로 해당 상태를 부모 행에서 확정
- `대습상속인 없음` 확정 시 우측 `대습상속 >>` 버튼은 숨긴다
