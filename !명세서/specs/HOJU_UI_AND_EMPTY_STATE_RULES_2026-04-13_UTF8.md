# 호주 선택 UI 및 빈상태 안내 규칙

작성일: 2026-04-13

## 목적

호주상속 가산 관련 입력과 안내를 사람별 땜질이 아니라 탭 기준 규칙으로 정리한다.

이 문서는 다음 두 축의 SSOT다.

1. 현재 탭 피상속인의 `호주/비호주` UI 규칙
2. 하위 상속인이 없을 때의 빈상태 안내 우선순위 규칙

## 핵심 원칙

1. 호주 여부는 사람 전역 속성이 아니라 상속 단계(탭) 기준으로 본다.
2. 원호주상속인은 현재 탭의 1차 상속인 중 1명만 지정한다.
3. 선사망/대습 관련 즉시 제지는 슬라이드 경고가 맡고, 빈상태 설명은 단일 안내 함수가 맡는다.
4. 빈상태 안내는 한 번에 1개만 출력한다.
5. 안내 박스 색상은 중성 그레이 톤으로 통일한다.

## 호주 UI 규칙

### 1. 카드 상단 호주 토글 노출 조건

현재 탭 피상속인 카드에서 `호주/비호주` 뱃지 토글을 보여주는 경우:

- 루트 피상속인
- 구법 단계의 `아들`
- 구법 단계의 `남편`

보여주지 않는 경우:

- `딸`
- `처`
- `1991년 이후 배우자`
- 일반 형제자매/직계존속 탭

### 2. 카드 상단 호주 토글 형태

- 별도 라벨 없이 작은 뱃지 버튼만 표시
- 기본 표시:
  - `호주`
  - 눌렀을 때 `비호주`
- 공간 절약을 위해 체크박스와 별도 설명 라벨은 두지 않는다.

### 3. 기본 정규화 규칙

AI/파일 불러오기 시:

- 구법 단계의 `아들`, `남편`은 기본적으로 호주 후보로 본다.
- AI JSON의 기계적 `isHoju: false` 값은 그대로 신뢰하지 않는다.
- 특별한 사정이 있으면 사용자가 탭에서 `비호주`로 명시적으로 바꾼다.

## 원호주상속인 규칙

1. 현재 탭 피상속인이 호주일 때만 `원호` 선택 UI를 보여준다.
2. 현재 탭의 1차 상속인 중 1명만 지정한다.
3. 이미 사망한 사람도 지정할 수 있다.
4. `원호`가 없더라도 같은 탭의 1차 상속인 중 `호주`로 체크된 사람이 있으면 우선 인정한다.

## 빈상태 안내 규칙

### 목적

하위 상속인이 아직 없을 때, 중복 배너 없이 현재 탭 기준으로 가장 중요한 메시지 1개만 보여준다.

### 구현 원칙

- `getEmptyStateGuide()` 같은 단일 함수가 메시지를 결정한다.
- 반환값은 `title`, `body` 한 쌍으로 충분하다.
- JSX 안에서 조건문을 계속 덧대지 않는다.

### 우선순위

1. 기본정보 미입력
2. 선사망 + 대습상속인 미입력
3. 구법 `아들/남편` 탭이 수동으로 `비호주` 지정됨
4. 구법 `루트/아들/남편` 탭으로서 호주 검토가 필요한 상태
5. 배우자 탭 자동분배 안내
6. 일반 자동분배 안내

### 권장 문구

#### 기본정보 미입력

- 제목: `기본정보를 먼저 입력해 주세요.`
- 본문: `사건번호, 피상속인 이름, 사망일자를 입력하면 다음 안내가 이어집니다.`

#### 선사망 + 대습상속인 없음

- 제목: `선사망 상속인은 대습상속인 입력이 필요합니다.`
- 본문: `배우자 또는 직계비속을 입력하지 않으면 계산에서 제외됩니다.`

#### 비호주 수동 지정

- 제목: `{현재 상속인명}은(는) 비호주로 설정되어 있습니다.`
- 본문: `특별한 사정이 없다면 호주 여부를 다시 확인한 뒤 1차 상속인을 입력해 주세요.`

#### 구법 호주 검토 필요

- 제목: `이 상속 단계는 호주상속 가산 검토가 필요합니다.`
- 본문: `상속인 불러오기 또는 수동 입력을 먼저 진행한 뒤, 현재 탭에서 호주 여부를 확인하세요.`

#### 배우자 탭 자동분배

- 제목: `별도의 상속인을 입력하지 않으면 [자녀들]에게 상속지분이 분배됩니다.`
- 본문: `자동 분배를 차단하려면 상속인을 추가해 주세요.`

#### 일반 자동분배

- 제목: `별도의 상속인을 입력하지 않으면 직계존속 또는 형제자매에게 상속지분이 분배됩니다.`
- 본문: `자동 분배를 차단하려면 상속인을 추가해 주세요.`

## 현재 구현 상태

반영 완료:

1. 상단 카드 `호주/비호주` 슬림 토글
2. 원호주상속인 선택 UI
3. 빈상태 안내 단일 함수화
4. 원호 미지정 안내와 빈상태 안내의 중복 제거
5. 안내 박스 그레이톤 통일

후속 구현 예정:

1. 사건형 호주가산 배너
2. `미반영시 결과보기` 비교 UI
3. 실제 가산 영향자 파란 강조
