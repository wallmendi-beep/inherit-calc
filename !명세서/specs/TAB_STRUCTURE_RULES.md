# 탭 구성 규칙 SSOT

작성일: 2026-05-01  
최종 수정: 2026-05-04  
관련 파일: `src/App.jsx`, `src/components/TreePanel.jsx`, `src/components/AcquisitionSumPanel.jsx`, `src/components/SummaryPanel.jsx`

---

## 1. 탭 목록

| 순서 | 탭 ID | 탭 라벨 | 한 줄 역할 |
|---|---|---|---|
| 1 | `input` | 데이터 입력 | 사실관계 원본 구축 |
| 2 | `tree` | 사건 검토 | 피상속인별 상속지분 분기와 사건별 판정 검토 |
| 3 | `summary` | 상속지분 | 취득합산, 취득계보, 최종지분표 검산 |

구 `acquisition` 탭은 독립 탭에서 제거한다. 취득경로 성격의 정보는 `summary` 탭 안의 `취득합산` 보기와 좌측 `취득계보` 패널이 담당한다.

## 2. 탭별 역할

### 2.1 `input` — 데이터 입력

**포함되어야 할 내용**
- vault 입력/수정
- 사람, 관계, 날짜, 호주/가적, 제외 사유 입력
- 좌측 가계도 요약 사이드패널

**포함되면 안 되는 내용**
- 계산 결과표
- 취득합산표
- 사건 보고서

---

### 2.2 `tree` — 사건 검토

**포함되어야 할 내용**
- 상속지분 분기 마인드맵
- 좌측 사건 보고서 패널
- 선택한 노드가 속한 상속사건의 사건 관계인, 사건 요약, 판정 요약
- 접기/펼치기, 전체 접기/펼치기, 선택 계통 집중 보기

**포함되면 안 되는 내용**
- 데이터 직접 입력 UI
- 최종 취득자별 합산표
- 상속권 없는 사람의 그래프 노드 표시
- 사건 보고서 안의 취득경로 요약

### 2.2.1 사건 검토 레이아웃 원칙

| 영역 | 역할 |
|---|---|
| 좌측 패널 | 사건 보고서 |
| 우측 영역 | 상속지분 마인드맵 |

사건 보고서는 선택한 노드 하나의 설명이 아니라, 그 노드가 속한 **하나의 상속사건**에 대한 판정 문서다.

### 2.2.2 사건 보고서 표시 규칙

기본정보는 두 줄로 압축한다.

```text
피상속인 김명수   상속지분 1분의 1
사망일자 1967년 10월 27일   적용법 구민법
```

적용법 표기는 다음 3개로 고정한다.

- `구민법`
- `79년 민법`
- `현행민법`

본문 섹션은 다음으로 제한한다.

1. `사건 관계인`
   - 컬럼: `성명 / 관계 / 지분 / 가감산 요소`
   - 지분은 `2/165`처럼 분수로 표시한다.
   - `판정` 컬럼은 두지 않는다.
   - 상속권이 없는 사람은 그래프 노드에서는 숨기되, 보고서에는 `상속권 없음`으로 표시한다.

2. `사건 요약`
   - `상속권 없음`, `가감산 요소`, `재상속·대습상속`으로 분리한다.
   - 입력된 사망일, 혼인일, 재혼일, 출가일 등 날짜가 있으면 문구에 반영한다.

3. `판정 요약`
   - 표와 사건 요약만으로 부족한 경우에만 짧게 설명한다.

---

### 2.3 `summary` — 상속지분

**포함되어야 할 내용**
- 좌측 취득계보 패널
- 우측 `취득합산` 표
- `최종지분표` 보기
- 구체적 상속분 산정 접기/펼치기 영역

**기본 보기**
- `summaryViewMode = 'sum'`
- 기본 화면은 `취득합산`이다.
- `최종지분표`는 토글로 전환한다.

**레이아웃 원칙**

| 영역 | 역할 |
|---|---|
| 좌측 패널 | 선택 상속인의 취득계보 |
| 우측 영역 | 취득합산표 또는 최종지분표 |

`summary` 탭은 `tree` 탭과 같은 좌측 패널/우측 작업 영역 구조를 따른다. 탭 전환 시 화면이 흔들리지 않도록 `input`용 가계도 요약 사이드패널은 숨긴다.

### 2.3.1 취득계보 패널

좌측 취득계보 패널은 선택한 최종 상속인의 지분이 최초 피상속인부터 누구를 거쳐 내려왔는지를 보여준다.

카드에는 다음 정보만 표시한다.

- 이름
- 지분

카드 사이에는 화살표와 흐름 라벨만 표시한다.

```text
김혁조        1/1
      ↓ 원상속
구수명        2/15
      ↓ 재상속
윤우영        1382/19305
      ↓ 재상속
윤성혜        2764/289575
```

카드 안에는 관계, 적용법, 사건명 설명을 넣지 않는다. 그 정보는 사건 검토 탭의 사건 보고서가 담당한다.

복수경로일 때는 `경로 1`, `경로 2`로 나누어 좌측 패널에만 표시한다.

### 2.3.2 취득합산표

우측 취득합산표는 숫자 검산만 담당한다.

컬럼은 다음으로 고정한다.

| 컬럼 | 내용 |
|---|---|
| `상속인` | 이름만 표시 |
| `경로합산` | 각 경로 지분의 숫자 합산 |
| `최종지분` | 합산 결과 |
| `통분지분` | 전체 최종 상속인 공통 분모 기준 지분 |

표 안에 넣지 않는다.

- 관계 표시
- 사건명 나열
- `단일경로`/`복수경로` 배지
- `김혁조→구수명→...` 같은 경로 문자열

경로 설명은 좌측 취득계보 패널만 담당한다.

---

## 3. 사이드패널 규칙

| 탭 | 좌측 패널 |
|---|---|
| `input` | 가계도 요약 |
| `tree` | 사건 보고서 |
| `summary` | 취득계보 |

전역 가계도 요약 패널은 `input` 탭에서만 표시한다.

---

## 4. 스마트 가이드 하이라이트

가이드 클릭 시 하이라이트는 위치 안내 수준으로 유지한다.

- 강한 파란 테두리 금지
- 확대 효과 금지
- 강한 그림자 금지
- 연한 배경과 얇은 회청색 테두리 사용

기준:

```text
background: #f7fbff
outline/shadow: #d7e5f9 2px
duration: 약 1.8초
```

---

## 5. 스마트 가이드 → 탭 워프 규칙

| 가이드 유형 | 목표 탭 | 이동 방식 |
|---|---|---|
| initial-step, struct-err, missing-death-date 등 | `input` | targetNodeIds 하이라이트 |
| missing-hoju, chained-hoju, legacy-stepchild, grouped-missing-substitution, grouped-direct-missing 등 | `tree` | navigationMode: `event` |
| import 이슈 (legacyHojuInputCase) | `tree` | navigationMode: `event` |
| import 이슈 (일반) | `input` | navigationMode: `auto` |

가이드는 `summary` 탭으로 직접 워프하지 않는다. 입력 검토는 `input`, 사건 판단 검토는 `tree`에서 처리한다.

---

## 6. 금지 패턴

- 독립 `acquisition` 탭 재도입 금지
- `summary` 우측 표에 경로 문자열 표시 금지
- `summary` 우측 표에 관계/사건명/단일경로/복수경로 표시 금지
- `tree` 사건 보고서에 취득합산표 추가 금지
- 같은 정보를 좌측 패널과 우측 표에 중복 표시 금지
