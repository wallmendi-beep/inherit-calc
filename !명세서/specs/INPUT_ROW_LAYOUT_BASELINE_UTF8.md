# 입력행 레이아웃 기준값

작성일: 2026-04-12

## 목적

데이터 입력 탭의 상속인 행([HeirRow.jsx](/C:/VS_CODE/상속지분%20계산기/src/components/HeirRow.jsx))과
헤더 행([InputPanel.jsx](/C:/VS_CODE/상속지분%20계산기/src/components/InputPanel.jsx))의 현재 정렬 상태를
기준값으로 고정한다.

향후 UI 미세조정 시 이 문서를 복원 기준으로 사용한다.

## 현재 기준 철학

- 윈도우 PC 와이드 모니터 기준의 실무형 정렬을 유지한다.
- 이름, 관계, 사망일, 특수조건, 재상속/지분, 삭제가 한 줄에서 안정적으로 보여야 한다.
- 배우자 선사망 경고, 재상속 버튼, 전체삭제 휴지통 위치는 현재 스냅샷을 기준으로 한다.
- 작은 간격 수정은 가능하지만, 컬럼 구조를 무너뜨리는 임의 확장은 금지한다.

## HeirRow 기준값

파일: [HeirRow.jsx](/C:/VS_CODE/상속지분%20계산기/src/components/HeirRow.jsx)

- 상태 토글 칸: `ml-[20px]`
- 이름 블록: `ml-[50px]`, `w-[72px]`
- 관계 칸: `ml-[30px]`, `w-[76px]`
- 생존/사망 입력 칸: `ml-[30px]`, `w-[150px]`
- 특수조건/가감산 칸: `ml-[10px]`, `w-[180px]`
- 재상속/지분 칸: `ml-[30px]`, `w-[88px]`
- 삭제 버튼 칸: `ml-[30px]`, `mr-[20px]`, `w-12`
- 호주 배지 위치: 이름 칸이 아니라 `특수조건 및 가감산` 칸

## InputPanel 헤더 기준값

파일: [InputPanel.jsx](/C:/VS_CODE/상속지분%20계산기/src/components/InputPanel.jsx)

- 상태 헤더: `ml-[20px]`, `w-7`
- 성명 헤더: `ml-[50px]`, `w-[72px]`
- 관계 헤더: `ml-[30px]`, `w-[76px]`
- 생존/사망(사망일자) 헤더: `ml-[50px]`, `w-[150px]`
- 특수조건 및 가감산 헤더: `ml-[10px]`, `w-[180px]`
- 재상속/지분 헤더: `ml-[10px]`, `w-[88px]`
- 전체삭제 휴지통 헤더: `ml-[20px]`, `mr-[20px]`, `w-12`

## 현재 UX 규칙

- 배우자 선사망은 `재상속 >>` 또는 `대습상속 >>` 버튼이 나타나면 안 된다.
- 배우자 선사망 토글을 강제로 켜면 경고 슬라이드가 뜨고 약 3초 후 자동 복귀한다.
- 선사망 배우자는 프런트에서도 `predeceased`로 유지되어야 한다.
- 호주 배지는 현재 기준에서 `특수조건 및 가감산` 열 소속으로 본다.

## 변경 원칙

- 다음 수정은 가능하면 `margin` 수치 땜질보다 `grid` 전환을 우선 검토한다.
- 단, 현재 스냅샷을 깨는 변경을 할 때는 이 문서를 먼저 갱신한다.
