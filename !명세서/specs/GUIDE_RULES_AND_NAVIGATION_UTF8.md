# 스마트 가이드 규칙 및 이동 기준

본 문서는 상속지분 계산기의 스마트 가이드가 언제 뜨는지, 어떤 문구를 쓰는지, 클릭 시 어디로 이동하는지를 정리한 운영 기준 문서다.

## 1. 가이드 분류

- `필수`
  계산 결과나 법리 판단에 직접 영향을 주는 입력 누락 또는 충돌
- `권고`
  계산은 가능하지만 법리적 안전성이나 입력 정합성을 위해 확인이 필요한 항목
- `참고`
  자동 분배, 결과 해석, 보조 설명

## 2. 표시 우선순위

1. 피상속인 기본정보 누락
2. 사망일 누락
3. 선사망/후사망 오판 가능성
4. 중복 배우자
5. 구법 호주 지정 필요
6. 권고 가이드

## 3. 대표 가이드 규칙

### 3.1. 피상속인 기본정보 누락

- 조건
  - 피상속인 이름 누락
  - 피상속인 사망일 누락
- 문구 예시
  - `사건번호와 피상속인 기본정보를 먼저 입력해 주세요.`
- 이동 위치
  - `root`

### 3.2. 사망자로 표시했는데 사망일이 없음

- 조건
  - `isDeceased = true`
  - `deathDate` 비어 있음
- 문구 예시
  - `[홍길동]은 사망자로 표시되어 있으나 사망일이 없습니다.`
- 이동 위치
  - 해당 인물 입력 위치

### 3.3. 선사망 자녀/형제자매인데 하위 상속인이 없음

- 조건
  - 피상속인보다 먼저 사망
  - 관계가 자녀 또는 형제자매
  - 하위 상속인 없음
- 문구 예시
  - `[홍길동]은 선사망자입니다. 대습상속인이 있으면 입력해 주세요.`
- 이동 위치
  - 해당 인물 탭

### 3.4. 선사망 배우자

- 조건
  - 배우자가 피상속인보다 먼저 사망
- 처리 원칙
  - 선사망 배우자는 별도 대습상속 안내를 띄우지 않는다.
  - 배우자 선사망은 `상속권 없음` 처리의 문제이지 `피대습자 입력` 문제가 아니다.
- 이동 위치
  - 없음

### 3.5. 중복 배우자

- 조건
  - 같은 기준 인물 아래 유효 배우자가 2명 이상 존재
- 문구 예시
  - `[홍길동] 기준 유효 배우자가 중복 입력되어 있습니다. 실제 상속받는 1명을 제외하고 정리해 주세요.`
- 이동 위치
  - 해당 기준 인물 탭

### 3.6. 구법 사건인데 호주 지정이 없음

- 조건
  - 구법 적용 사건
  - 직계비속 또는 대습상속 가지가 존재
  - 호주상속 지정 없음
- 문구 예시
  - `구법 적용 대상입니다. 하위 상속인 중 호주상속인을 확인해 주세요.`
- 이동 위치
  - 해당 피상속인 탭

### 3.7. 전역 불일치 경고

- 조건
  - 계산 결과 합이 1이 되지 않음
  - 사망자 지분이 최종 결과에 잔존
  - 재상속/대습상속 전달 누락 가능성
- 문구 예시
  - `전체 지분 합계가 일치하지 않습니다.`
- 이동 위치
  - 원인이 특정되면 해당 인물
  - 특정 불가 시 `root`

## 4. 클릭 이동 규칙

- 원칙
  - 클릭 시 수정이 필요한 탭으로 이동한다.
- 우선순위
  1. `targetTabId`
  2. `personId`
  3. `root`

## 5. 체크/숨김 규칙

- 권고 가이드는 사용자가 숨길 수 있다.
- 숨김은 영구 삭제가 아니라 `현재 같은 uniqueKey`의 재노출만 막는다.
- 조건이 실제로 해소되면 숨김 상태도 같이 의미를 잃는다.

## 6. 구현 원칙

- 가이드는 UI에서 임의 생성하지 않는다.
- 가능한 한 동일한 판정 소스를 계산/배지/가이드가 공유한다.
- 선사망 배우자는 대습상속 가이드 대상에서 제외한다.
- 필수 가이드와 권고 가이드는 반드시 구분해 렌더링한다.
