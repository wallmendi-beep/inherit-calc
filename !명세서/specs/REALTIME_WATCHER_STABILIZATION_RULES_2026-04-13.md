# 실시간 감시 안정화 규칙 (2026-04-13)

## 배경
- 대형 사건 편집 중 메모리 사용량이 시간이 지날수록 수 GB 단위로 증가하고, 최종적으로 앱이 멈추는 문제가 관찰되었다.
- 원인 후보로 실시간 검수 재실행과 스마트 가이드 전체 재계산이 지목되었다.

## 안정화 원칙
1. 입력 중에는 가능한 한 무거운 전트리 재검사를 반복하지 않는다.
2. 실시간 감시는 즉시성보다 안정성을 우선한다.
3. 사용자가 실제로 보는 순간에만 무거운 파생 계산을 수행한다.

## 현재 적용 규칙

### 1. importIssues 실시간 재검사 중단
- `importIssues`는 파일/AI 불러오기 시점에 한 번 생성한다.
- 입력 중 실시간으로 `collectImportValidationIssues(tree)`를 반복 실행하지 않는다.
- 사용자가 다시 불러오거나 새 입력을 시작할 때 갱신한다.

### 2. 스마트 가이드 지연 계산
- `useSmartGuide()`는 아래 경우에만 실제 계산한다.
  - 스마트 가이드 패널이 열려 있을 때
  - 현재 탭이 `input`일 때
- 그 외 탭에서는 빈 데이터 기준으로 동작시켜 전체 트리 순회 비용을 줄인다.

## 기대 효과
- 시간 경과에 따른 메모리 상승 완화
- 입력 중 버벅임 감소
- 장시간 편집 시 프리징 가능성 감소

## 후속 과제
1. 필요 시 `검수 실행` 버튼 기반 수동 검수 모델로 이동
2. 스마트 가이드 내부 순회 로직 자체를 더 분해
3. import 검수와 guide 검수의 기준 시점을 명시적으로 분리
