# 탭 역할 정의 및 가계도 시뮬레이션 초안

작성일: 2026-04-13  
적용 범위: `데이터 입력`, `가계도`, `계산 상세`, `결과 리포트`, `법정 상속분`, `구체적 상속분`

---

## 1. 목적

본 문서는 상속지분 계산기 각 탭의 역할을 다시 분명히 정의하고,  
특히 `가계도` 탭을 단순 트리 요약이 아니라 **지분 이동 시뮬레이션 탭**으로 확장하기 위한 초안을 정리한다.

핵심 목표는 다음과 같다.

1. `데이터 입력` 탭은 사실관계 입력에만 집중한다.
2. `가계도` 탭은 `vault -> tree` 변환 결과와 지분 이동 흐름을 시각화한다.
3. `계산 상세` 탭은 법리와 수치 계산의 테이블 검산 화면이 된다.
4. 사용자는 계산 결과를 보다가 사람 이름을 눌러 입력 탭으로 돌아가 수정할 수 있어야 한다.

---

## 2. 탭 역할 SSOT

### 2.1 데이터 입력

정의:
- `vault` 정리용 화면
- 사람, 관계, 날짜, 상태, 호주상속/재산상속, 동일가적/비동일가적 입력
- 정적인 입력과 수정의 원본 화면

이 탭에서 해야 하는 일:
- 피상속인 및 상속인 정보 입력
- 관계 정리
- 사망일, 혼인일, 재혼일, 복적일 입력
- 후속 상속인(재상속/대습상속) 추가
- 입력 오류 수정

이 탭에서 하지 않을 일:
- 최종 지분 확정값을 전면 표시하지 않음
- 복잡한 법리 설명을 장황하게 표시하지 않음
- 계산 결과 비교를 주역으로 두지 않음

한 줄 원칙:
- `데이터 입력 = 사실관계 원본 구축`

### 2.2 가계도

정의:
- `vault -> tree` 변환 결과 확인 화면
- 입력된 사람과 관계가 실제 계보 구조에서 어떻게 연결되는지 보는 화면
- 동시에, 사망 단계별 지분 이동 흐름을 시뮬레이션하는 화면

이 탭에서 해야 하는 일:
- 현재 계보 구조 요약
- 사망자별 상속 흐름 카드 제공
- `누가 죽으면서 누구에게 넘어갔는지`를 단계별로 시각화
- 대습상속 / 재상속 그룹을 카드 단위로 표현

이 탭에서 하지 않을 일:
- 최종 계산식을 표 형태로 장황하게 전부 나열하지 않음
- 입력폼처럼 세부 사실값 편집을 직접 수행하지 않음

한 줄 원칙:
- `가계도 = 구조 확인 + 지분 이동 시뮬레이션`

### 2.3 계산 상세

정의:
- `vault + tree + 계산엔진`을 결합한 법리/수치 검산 화면
- 상속권, 적용 규정, 계산식, 상속분, 비고를 확인하는 테이블 화면

이 탭에서 해야 하는 일:
- 누가 상속권을 가지는지 표시
- 어떤 규정이 적용되는지 표시
- 계산식과 최종 분수 표시
- 이름 클릭 시 입력 탭 해당 사람으로 이동

이 탭에서 하지 않을 일:
- 원본 데이터 직접 입력을 여기서 끝내지 않음
- 가계도 시각화 역할까지 떠안지 않음

한 줄 원칙:
- `계산 상세 = 법리와 수치 검산`

### 2.4 결과 리포트 / 법정 상속분 / 구체적 상속분

정의:
- 계산이 완료된 결과를 정리해 보여주는 결과층 화면

한 줄 원칙:
- `결과 탭 = 계산 완료본 정리와 보고`

---

## 3. 현재 구조 해석

현재 프로젝트 구조를 가장 정확히 표현하면 다음과 같다.

1. `데이터 입력`
   - `vault` 입력/수정
2. `가계도`
   - `vault -> tree` 구조 확인
3. `계산 상세`
   - `vault + tree + engine` 결합 결과
4. `결과 탭`
   - 최종 출력

즉 본 계산기는 더 이상 `입력 탭 하나가 모든 것을 처리하는 구조`가 아니며,  
`입력 / 구조 / 계산 / 결과`로 기능이 분화된 멀티레이어 구조로 보는 것이 맞다.

## 4. 가계도 탭 시뮬레이션 초안

### 4.1 방향

기존 가계도 탭은 단순 트리 요약에 가깝다.  
앞으로는 여기에 **사망자별 상속 흐름 카드**를 추가하여, 지분 이동 과정을 사용자에게 직관적으로 보여주도록 한다.

### 4.2 추천 UI 구조

상단:
- 현재 선택 계보 요약
- 검색 / 하이라이트
- 보기 모드 토글
  - `트리 보기`
  - `흐름 보기`

본문 1영역: 계보 트리
- 현재의 가족 구조를 트리로 표시
- 클릭 시 해당 사람 하이라이트

본문 2영역: 상속 흐름 카드
- 사망자별 카드 목록
- 예:
  - `김혁조 사망`
  - `구수명 사망`
  - `김일상 사망`
  - `김일주 사망`

각 카드 내부:
- 사망자 이름
- 사망일
- 당시 기준 시기
- 원지분
- 1차 분배 대상
- 선사망이면 `대습상속`
- 후사망이면 `재상속`
- 최종 귀착자 요약

### 4.3 카드 예시

#### 김혁조 사망

- 원지분: `1 / 1`
- 1차 분배:
  - 김명남 계통
  - 김명수 계통
  - 김말수 계통
  - 김일상 계통
  - 김일주 계통
- 특이사항:
  - 김일상: 호주상속 5할 가산

#### 김일상 사망

- 원지분: `74 / 165`
- 1차 분배:
  - 이경숙
  - 김경진
  - 김화경
  - 김지훈
  - 김진수
  - 김혜경
- 후속 흐름:
  - 이경숙 사망 -> 자녀 1/6 재상속
  - 김혜경 사망 -> 남편/자녀 3:2:2

### 4.4 시뮬레이션 카드 표시 원칙

1. 가계도 탭은 `최종 숫자 보고서`가 아니라 `흐름 시각화`에 집중한다.
2. 모든 상세 계산식은 계산 상세 탭이 우선이다.
3. 가계도 탭에서는 숫자를 최소화하되, 원지분과 핵심 분기만 보여준다.
4. 사람이름 클릭 시 해당 입력 탭 또는 계산 상세 위치로 이동할 수 있어야 한다.

---

## 5. 계산 상세와의 역할 분리

가계도 탭과 계산 상세 탭의 차이는 아래와 같이 유지한다.

가계도 탭:
- 누가 죽고 누구에게 넘어갔는지
- 상속 흐름의 시각화
- 단계별 카드/흐름 요약

계산 상세 탭:
- 왜 그렇게 계산됐는지
- 어떤 규정이 적용됐는지
- 수치가 정확히 얼마인지

즉:
- `가계도 = 흐름`
- `계산 상세 = 검산`

---

## 6. 구현 우선순위 제안

### 1차

- 가계도 탭에 `흐름 보기` 모드 추가
- `finalShares.subGroups`, `transitShares`, `calcSteps`를 사용해 사망자별 카드 생성

### 2차

- 카드 클릭 시 해당 사람 입력 탭 이동
- 카드 내 `재상속`, `대습상속` 뱃지 추가

### 3차

- 트리와 흐름 카드를 동기화
- 트리에서 사람 선택 시 해당 흐름 카드 하이라이트
- 흐름 카드 선택 시 트리 노드 하이라이트

---

## 7. 설계 결론

현재 프로젝트의 가장 적절한 탭 책임 정의는 다음과 같다.

- `데이터 입력`: vault 원본 정리
- `가계도`: tree 구조 확인 + 지분 이동 시뮬레이션
- `계산 상세`: vault/tree/engine 결합 결과 검산
- `결과 탭`: 최종 결과 정리

이 문서를 이후 가계도 탭 개편의 SSOT로 사용한다.
