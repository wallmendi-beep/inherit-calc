# v4.45 변경로그 기반 이력 경량화 1차

## 배경
- 대형 사건 편집 시 앱이 버벅이거나 장시간 후 응답없음처럼 보이는 문제가 있었다.
- 메모리 사용량이 수 GB 단위까지 커지는 현상이 관찰되었다.
- 원인 중 하나로 전체 `vault` 스냅샷 히스토리의 과도한 누적이 의심되었다.

## 이번 변경
1. `vaultState.history` 최대 길이를 `50 -> 10`으로 축소
2. `changeLog` 상태 추가
3. 주요 사용자 조작을 의미 단위 action log로 기록
4. `changeLog`를 `localStorage`에 디바운스 오토세이브
5. 깊은 복사 시 `structuredClone` 우선 사용
6. 호주가산 미반영 비교 계산을 `calc/result/summary` 탭에서만 수행
7. `importIssues` 재검사를 `input` 탭에서만 수행
8. `importIssues` 비교 로직을 `JSON.stringify` 전수 비교에서 `signature` 비교로 경량화

## 기대 효과
- 메모리 사용량 감소
- 수정 직후 멈칫거림 완화
- 장기적으로 action-based undo/redo 전환을 위한 기반 확보

## 한계
- 아직 undo/redo는 전체 action replay가 아니라 축소된 snapshot history 기반이다.
- `useSmartGuide`는 여전히 큰 사건에서 추가 최적화 여지가 있다.

## 후속 작업
1. `baseVault + actionLog + redoLog` 구조 고도화
2. `useSmartGuide` 순회 비용 절감
3. 복구용 임시 저장 구조 정교화
