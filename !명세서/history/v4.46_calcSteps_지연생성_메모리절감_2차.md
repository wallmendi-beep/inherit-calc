# v4.46 calcSteps 지연생성 메모리절감 2차

## 배경
- 변경로그/히스토리 경량화 1차 후에도 메모리 사용량이 수 GB 수준으로 높게 유지되었다.
- 원인 후보로 `calculateInheritance()`의 대형 `calcSteps` 생성 비용이 확인되었다.

## 이번 변경
1. `calculateInheritance()`에 `includeCalcSteps` 옵션 추가
2. 입력 탭에서는 `calcSteps`를 생성하지 않도록 변경
3. `호주가산 미반영 비교` 계산은 `calcSteps` 없이 수행

## 기대 효과
- 입력 탭에서 불필요한 상세 계산 스텝 메모리 제거
- 비교 계산 시 중복 상세 스텝 생성 방지
