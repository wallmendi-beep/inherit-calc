# 업데이트 노트 (v4.80) - 사건 맥락 기준 대습/재상속 가이드 수정

작성일: 2026-05-03

## 수정 배경

김혁조 사건 파일 검토 중 같은 인물인 정문자가 사건 흐름에 따라 다르게 판정되어야 하는 문제가 확인되었다.

- 김혁조 -> 김명수 사건에서는 김명수 사망 후 정문자가 사망했으므로 정문자는 재상속/후속상속 검토 대상이다.
- 구수명 사건에서는 구수명보다 정문자가 먼저 사망했으므로 정문자는 생존 상속인으로 표시되면 안 된다. 필요하면 김명수 계통의 대습상속인 입력 또는 없음 확정으로 정리해야 한다.

## 원인

v4.79에서 선사망/후사망 비교 기준을 직접 부모 사망일로 단순화했다. 이 방식은 단일 사건에서는 맞아 보이지만, 사건 검토 탭처럼 `한 사건 = 한 피상속인의 사망` 단위로 흐름을 나누는 구조에서는 구수명 가지의 사건 맥락을 잃는다.

## 반영 내용

1. `useSmartGuide.js`
   - 선사망/후사망 가이드 판정을 `contextDate/contextNode` 기반으로 변경
   - 후사망자를 만나면 그 사람의 사망일을 새 사건 기준일로 전환
   - 선사망자를 만나면 기존 사건 기준일을 유지
   - 대습상속 미확정 문구에 필요한 경우 `김명수 계통 선사망자`처럼 계통명을 표시

2. `importValidationV2.js`
   - import 검증도 같은 사건 맥락 전이 방식으로 변경
   - 선사망자는 generic `후속 상속 미확정` import issue를 만들지 않음
   - 선사망자는 SmartGuide의 대습상속 검토 카드가 담당

3. 문서
   - `PREDECEASE_REINHERITANCE_JUDGMENT_AND_WARP_RULES.md`를 사건 맥락 기준으로 전면 정리

## 기대 결과

- 김혁조 -> 김명수 흐름: 정문자는 후속상속 검토 대상
- 구수명 흐름: 정문자는 생존 상속인 카드에 나오지 않고, 필요하면 김명수 계통 선사망자 대습 검토로 안내
- 같은 사건 맥락 안에서 `대습상속 미확정`과 `후속 상속 미확정`이 중복 표시되는 문제 감소
