# v4.38 상속인 불러오기 서브트리 복제 및 순환참조 재발방지

## 배경
- `윤방자`, `윤우영` 등에서 순차상속 순환참조 경고가 나타났고,
- 빈상태 안내는 형제자매 자동분배를 제안했으나 실제로는 기존에 배우자/자녀가 이미 입력되어 있는 상태였다.

## 확인된 원인
- `appendResolvedHeirs()`가 상속인 불러오기 시 **하위 상속인 트리를 복제하지 않고 상위 인물 1명만 추가**하고 있었다.
- 이 때문에 사망자가 새 탭에서는 `하위 상속인 없음`처럼 보였고, 엔진이 형제자매 fallback을 타면서 순환참조 가능성이 커졌다.

## 수정 사항
- `src/App.jsx`
  - `appendResolvedHeirs()`를 재귀 복제 방식으로 변경
  - 불러온 인물의 배우자/자녀/하위 상속인 전체를 `vault`에 함께 복제

## 기대 효과
- 사망자 불러오기 후 하위 상속인 유실 방지
- 잘못된 형제자매 자동분배 안내 감소
- 동일 유형의 순환참조 재발 가능성 감소

## 검증
- `npm run build` 통과
- `npx vitest run src/engine/inheritance.test.js` 통과
