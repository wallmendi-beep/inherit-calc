# 업데이트 노트 (v4.85) - 후속상속 가이드 입력 탭 직접 이동

작성일: 2026-05-03

## 수정 배경

정문자 관련 가이드에서 사용자가 실제로 처리해야 하는 위치는 정문자 하위 상속인 입력 탭이다. 그러나 `후속 상속 미확정` 카드가 사건 검토 이동으로 처리되거나, 중복 personId 탐색 때문에 기대한 정문자 탭으로 가지 않는 문제가 있었다.

## 반영 내용

1. 후속상속 미확정 카드는 `navigationMode: input`으로 분리
   - 클릭 시 데이터 입력 탭으로 이동
   - `targetTabId`의 본인 사건 탭을 직접 연다
   - 단일 대상이면 실제 `node.id`도 함께 전달한다

2. 같은 인물이 후속상속 미확정 대상이면 대습상속 미확정 목록에서는 제외
   - 정문자처럼 구수명 사건에서는 대습 검토 대상이지만 김명수 사건에서는 후속상속 검토 대상인 경우, 사용자 액션은 정문자 하위 상속인 입력으로 동일하다.
   - 따라서 후속상속 카드를 우선하고 대습상속 카드의 중복 이름은 숨긴다.

## 기대 결과

- `후속 상속 미확정 — [김명수] 사건: [정문자]` 클릭 시 정문자 입력 탭으로 이동한다.
- 정문자가 후속상속 카드에 남아 있는 동안 `대습상속 미확정 — [구수명] 사건의 김명수 계통 선사망자: [정문자]` 카드는 표시되지 않는다.
