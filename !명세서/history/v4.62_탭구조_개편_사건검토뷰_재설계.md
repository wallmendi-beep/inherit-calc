이번 버전에서는 **탭 구조 전반을 재검토**하고, 오랫동안 누적된 설계 불일치를 해소했다.

3. **TreePanel 전면 재설계**: 카드 나열 방식 → 좌측 네비게이터 + 우측 캐스케이드 뷰

## 2. 배경 및 설계 철학

### 2.1 왜 이 작업이 필요했나

이 탭(구: 시뮬레이션)의 진화 과정:

```
원래 목적:
  순수 가계도 확인 → 상속인 누락 방지

문제 발생:
  김혁조 같은 복잡한 사건은 가계도만으로 검증 불가
  → 사망 시점 23년 차이, 같은 인물의 다른 법적 조건

해결:
  가계도 역할 → 사이드바(가계도 요약)에 이관
  이 탭 → 피상속인 중심의 입력 검증 탭으로 재설계
```

### 2.2 사건 분리 철학

복잡한 상속 연쇄를 한 화면에서 보면 인간의 인지 한계를 초과한다.  
**`한 사망 사건 = 독립된 검토 단위`** 원칙으로 쪼개어,  
각 사건을 별도로 확인하되 재상속/대습상속은 해당 사건으로 이동 버튼만 제공한다.

이 원칙은 `calcSteps` 데이터 구조와 정확히 일치한다 — 엔진은 이미 이렇게 계산하고 있었다.  
문제는 **UI가 그 구조를 제대로 표현하지 못했던 것**이다.

### 2.3 `result` + `summary` 통합 근거

| 뷰 | 기존 탭 | 설명 |
|----|---------|------|
| 지분 구조 | 법정 상속분 | 가계도 구조 기준, 그룹화, 통분 검증 |
| 취득 경로 | 결과 리포트 | 상속인별 취득 경로 역추적 |

두 뷰는 같은 데이터(`finalShares`, `calcSteps`)를 **다른 정렬 방식**으로 보여줄 뿐이다.  
사용자도 두 탭의 차이를 명확히 설명하기 어려웠고, 비슷한 메뉴가 두 개인 것이 불편했다.  
→ 내부 토글(지분 구조 / 취득 경로)로 통합, 탭 1개 절감.

- 검색 인풋은 `지분 구조` 뷰에서만 표시

### 3.3 TreePanel.jsx (전면 재설계)

#### 삭제된 것
- `EventListItem` 컴포넌트 (기존 사이드바 아이템)
- `RecipientCard` 컴포넌트 (기존 카드)
- `FlowNode`, `FlowTag`, `SummaryPill` 컴포넌트
- 3중 중첩 카드 구조

#### 추가된 것

**`NavItem`** — 사건 네비게이터 아이템
- 상속 체인 depth 반영한 들여쓰기
- 번호 뱃지 (선택시 파란색, 비선택시 회색)
- `↓` 표시로 자식 사건 존재 여부 안내

**`DirectionBanner`** — 상속 방향 배너
- 적용 민법 표시 (구민법 / 1979년 개정 / 현행)
- 배우자 동순위 여부
- 상속 순위 (1순위 직계비속 / 2순위 직계존속 / 3순위 형제자매)
- 상속지분 표시

**`ExcludedSection`** — 제외된 상속인 목록
- 상속권 없는 인물 + 제외 이유를 취소선 + 뱃지로 표시
- 예: `조성제 ~~조성제~~ [1991년 이전 처 사망으로 대습불가]`

**`HeirCard`** — 상속인 카드 (재설계)
- 배우자: 좌측 파란색 accent bar
- 비배우자: 좌측 중립 accent bar
- 계산식: `상속지분 × 내부지분` → `최종지분` (크게 표시)
- 특수조건 뱃지: 가산/감산/호주상속/출가녀 등
- 재상속/대습상속 사건 링크 버튼

**`buildStepTree()`** — 사건 간 부모-자식 관계 맵 빌드 함수
- `parentOf`: 자식 stepKey → 부모 stepKey
- `childrenOf`: 부모 stepKey → 자식 stepKey 배열
- 네비게이터 depth 계산에 사용

#### 레이아웃 구조
```
[모드 전환 바]
  └── [트리 보기] / [사건 검토] 토글

[사건 검토 모드]
  ├── [좌측 200px: 사건 네비게이터]
  │    └── NavItem × N (depth 반영)
  └── [우측: 사건 상세 캐스케이드]
       ├── 피상속인 헤더
       ├── DirectionBanner
       ├── ExcludedSection
       ├── 배우자 섹션 (HeirCard)
       ├── 구분선
       └── 직계비속/존속/형제 섹션 (HeirCard grid)
```
