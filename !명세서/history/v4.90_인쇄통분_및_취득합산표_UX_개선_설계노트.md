# 업데이트 노트 (v4.90) - 인쇄 통분 및 취득합산표 UX 개선 설계노트

작성일: 2026-05-07

## 1. 배경

김혁조 사건 검토 중 화면에서는 반영된 그룹 통분과 취득합산 UX가 인쇄 화면 또는 상속지분 탭 일부 영역에서는 일관되게 반영되지 않는 문제가 확인되었다.

또한 `김문섭 비실명 테스트 데이터`로 진행하던 익명화 프로젝트는 실패한 프로젝트로 판단하여 폐기한다. 이후 검토 기준에서 익명화 명세와 비실명 fixture를 제외한다.

폐기 대상:

- `!명세서/specs/KIM_HYEOKJO_ANONYMIZED_TEST_DATA_2026-05-07.md`
- `src/utils/kimHyeokjoAnonymizedTestData.js`
- `VITE_TEST_DATA=kimhyeokjo`
- `김문섭` 비실명 JSON, 치환표, 익명화 검증 원칙

## 2. 인쇄 화면 그룹 통분 누락

### 2.1 사건검토 탭 인쇄 산출표

사건검토 탭의 인쇄 화면에서 제4사건 윤우영 사건 예시가 다음처럼 표시되었다.

```text
제4사건 망 윤우영 2013년 4월 26일 사망 분배 지분 9788/353925 현행 민법

김경희(A) 배우자 9788/353925 × 3/9 9788/1061775 처(배우자) 5할 가산
윤성혜     자녀   9788/353925 × 2/9 19576/3185325 균분
윤상호     자녀   9788/353925 × 2/9 19576/3185325 균분
윤정호     자녀   9788/353925 × 2/9 19576/3185325 균분
```

계산식의 내부 분배비율은 `3/9`, `2/9`로 통분되어 있으나, `취득 지분` 칸은 엔진의 약분 결과를 그대로 출력한다.

문제 위치:

- `src/components/PrintReport.jsx`
  - 사건검토 인쇄 표에서 `innerLCM`은 계산하지만 `취득 지분`은 `{d.n}/{d.d}`를 그대로 출력한다.

수정 기준:

- 사건 내부 활성 분배자 기준으로 취득 지분의 공통분모를 계산한다.
- `취득 지분` 칸도 같은 사건/가족 그룹 안에서 통분된 값으로 표시한다.
- 윤우영 사건은 다음처럼 보여야 한다.

```text
김경희(A) 29364/3185325
윤성혜    19576/3185325
윤상호    19576/3185325
윤정호    19576/3185325
```

### 2.2 최종지분표 인쇄

화면의 최종지분표는 `최종 지분`, `그룹통분 지분`, `전체통분 지분`을 구분해 표시한다.

그러나 인쇄용 요약표는 현재 다음 3개 컬럼만 가진다.

```text
상속인 | 법정 지분 | 법정 지분(통분)
```

이 구조에서는 화면의 `그룹통분 지분`이 인쇄되지 않는다.

수정 기준:

```text
상속인 | 최종 지분 | 그룹통분 지분 | 전체통분 지분
```

- 직접 상속 그룹도 그룹 공통분모를 계산해 표시한다.
- 재상속·대습상속 그룹은 그룹 헤더별 직계 행끼리 공통분모를 계산해 표시한다.
- `전체통분 지분`은 기존 `un/ud` 값을 유지한다.

### 2.3 인쇄 모드 상태 전달

현재 `App.jsx`에서 `PrintReport`에 `summaryViewMode`가 전달되지 않는다.

따라서 상속지분 탭에서 사용자가 `취득합산` 또는 `최종지분표` 중 무엇을 보고 있는지 인쇄 컴포넌트가 알 수 없다.

수정 기준:

- `PrintReport` 호출 시 `summaryViewMode={summaryViewMode}`를 전달한다.
- 상속지분 탭 인쇄는 현재 선택된 하위 보기와 일치해야 한다.
- `취득합산` 인쇄와 `최종지분표` 인쇄의 표 구조를 분리한다.

## 3. 상속지분 탭 타이포그래피 조정

상속지분 탭의 기본 화면인 `취득합산`은 표 본문 대부분이 `font-bold` 또는 `font-black`으로 지정되어 있어 그래픽적으로 과하다.

확인된 과도한 강조:

- 상속인 이름: `font-black`
- 경로합산: `font-bold`
- 최종지분: `font-black`
- 통분지분: `font-black`
- 좌측 선택자 이름과 지분: `font-black`

조정 기준:

| 영역 | 권장 굵기 |
|---|---|
| 표 본문 기본 | `font-normal` 또는 `font-medium` |
| 상속인 이름 | `font-medium` |
| 경로합산 | `font-normal` 또는 `font-medium` |
| 최종지분·통분지분 | `font-semibold` |
| 제목·선택자 핵심값 | 제한적으로 `font-bold` |
| 오류·불일치·현재 선택 강조 | 제한적으로 `font-black` 또는 색 강조 |

목표는 모든 숫자를 강하게 보이게 하는 것이 아니라, 선택 행과 최종 결론만 자연스럽게 눈에 띄도록 하는 것이다.

## 4. 취득합산표 구조 정리

### 4.1 역할 정의

`취득합산표`의 역할은 사건검토 탭에서 사건별로 분배된 상속지분을 최종 상속인별로 수렴해 보여주는 것이다.

따라서 표 전체를 계통별로 재배열하지 않는다. 행의 주인은 최종 상속인이다.

역할 구분:

| 화면 | 역할 |
|---|---|
| 사건검토 탭 | 사건별 분배 과정 표시 |
| 취득합산표 | 사건별 취득분을 최종 상속인별로 수렴·합산 |
| 최종지분표 | 최종 생존 상속인별 지분을 그룹통분·전체통분 기준으로 검토 |
| 좌측 취득계보 패널 | 선택한 상속인의 경로별 취득 흐름 설명 |

### 4.2 컬럼 유지

새 컬럼 `취득원천`은 추가하지 않는다.

현재 구조를 유지한다.

```text
상속인 | 경로합산 | 최종지분 | 통분지분
```

단, `경로합산`의 표시 방식을 개선한다.

### 4.3 경로합산 표시

현재 `경로합산`은 분수 합산식을 보여주는 칸이어야 한다.

사람 이름 경로는 이 칸에 넣지 않는다. 사람 이름의 흐름은 좌측 `취득계보` 패널에서 표시한다.

권장 표시:

```text
① 29364/3185325 + ② 10/300
```

원칙:

- `경로 1`, `경로 2` 대신 동그라미 번호 `①`, `②`를 사용한다.
- 사건명, 망인명, 계통명은 표 안에 길게 넣지 않는다.
- 표에서는 숫자 합산식을 우선한다.
- 동그라미 번호는 좌측 패널의 해당 경로와 연결된다.

### 4.4 레이아웃

상속인 칸은 현재 여유가 크므로 폭을 줄인다.

권장:

| 컬럼 | 폭 |
|---|---|
| 상속인 | 약 `120px` 고정 |
| 경로합산 | 남은 폭 대부분 |
| 최종지분 | 약 `160px` |
| 통분지분 | 약 `160px` |

이렇게 하면 `경로합산`을 세로로 쌓지 않고 한 줄로 표시할 가능성이 커진다.

긴 합산식은 우선 한 줄 표시를 기본으로 하고, 불가피한 경우 말줄임 처리 후 좌측 패널에서 상세 경로를 확인하게 한다.

## 5. 좌측 패널: 취득계보

### 5.1 패널 이름

취득합산표 좌측 패널의 이름은 `취득계보`로 정한다.

이 패널은 최종 지분 자체보다, 최초 피상속인부터 선택자까지 내려온 취득 경로를 설명하는 영역이다.

후보 중 `취득계보`를 채택한 이유:

- 짧고 명확하다.
- 현재 설명문인 "최초 피상속인부터 선택자까지 내려온 취득 계보"와 맞다.
- `취득 경로`보다 여러 경로가 합쳐지는 의미를 더 잘 담는다.

### 5.2 경로 번호 표시

좌측 패널의 경로 제목도 취득합산표와 맞춘다.

권장 표시:

```text
① 경로
② 경로
```

표에서는:

```text
① 29364/3185325 + ② 10/300
```

좌측 패널에서는:

```text
① 경로
김혁조
↓ 재상속
김명남
↓ 재상속
윤우영
↓ 최종 취득
김경희(A)
```

### 5.3 최종 취득자 강조

현재 취득계보의 각 카드가 비슷한 시각 무게로 표시되어 최종 지분취득자가 잘 눈에 띄지 않는다.

수정 기준:

- 중간 경로 카드는 보통 굵기와 옅은 테두리로 표시한다.
- 마지막 카드만 결론 카드처럼 강조한다.
- 마지막 카드에는 `최종 취득` 배지를 붙인다.
- 마지막 카드의 이름과 지분만 한 단계 굵게 표시한다.

권장 스타일:

| 영역 | 표시 |
|---|---|
| 중간 카드 | 흰 배경, 옅은 테두리, `font-medium` |
| 최종 카드 | 파란 계열 배경, 왼쪽 강조선, `최종 취득` 배지 |
| 마지막 직전 화살표 | `↓ 최종 취득` |

## 6. 회귀 검증 포인트

### 6.1 사건검토 인쇄

윤우영 사건 인쇄 표에서 배우자와 자녀가 같은 분모로 표시되어야 한다.

```text
김경희(A): 29364/3185325
윤성혜:    19576/3185325
윤상호:    19576/3185325
윤정호:    19576/3185325
```

### 6.2 최종지분표 인쇄

최종지분표 인쇄에 다음 컬럼이 모두 표시되어야 한다.

```text
상속인 | 최종 지분 | 그룹통분 지분 | 전체통분 지분
```

화면 최종지분표와 인쇄 최종지분표의 그룹통분 값이 일치해야 한다.

### 6.3 상속지분 탭 하위 보기 인쇄

상속지분 탭에서 `취득합산`을 선택한 상태로 인쇄하면 취득합산표가 인쇄되어야 한다.

상속지분 탭에서 `최종지분표`를 선택한 상태로 인쇄하면 최종지분표가 인쇄되어야 한다.

### 6.4 취득합산표

- 상속인 칸이 약 `120px`로 줄어든다.
- 경로합산 칸은 `① 분수 + ② 분수` 형식으로 표시된다.
- 경로합산 칸에 사람 이름 경로를 넣지 않는다.
- 좌측 취득계보 패널의 `① 경로`, `② 경로`와 표의 동그라미 번호가 대응한다.

### 6.5 취득계보 패널

- 패널 제목이 `취득계보`로 표시된다.
- 마지막 카드가 `최종 취득`으로 명확하게 강조된다.
- 중간 카드와 최종 카드의 시각 무게가 구분된다.
