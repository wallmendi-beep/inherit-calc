# v4.34 상속지분 계산룰 문서화 및 출처별 분배근거 보강

## 변경 목적
- 김혁조 사건처럼 `부(김혁조)분 + 모(구수명)분`이 합산되는 사건에서,
  단순 최종 지분만으로는 계산 근거를 추적하기 어려운 문제를 보완한다.
- 신청인 계산표와 엔진 결과를 비교할 때, 어떤 단계에서 차이가 나는지 역추적할 수 있도록 한다.
- 상속지분 계산 법리 자체를 별도 명세로 분리하여, 이후 Gemini/Codex가 읽고 판단하기 쉽게 만든다.

## 주요 변경 사항

### 1. 계산룰 전용 명세서 추가
- 신규 문서:
  - `!명세서/specs/INHERITANCE_SHARE_RULES.md`
- 포함 내용:
  - 시점별 상속 법 적용 원칙
  - 1960~1978 / 1979~1990 / 1991~현재 상속분 규칙
  - 배우자 대습 규칙
  - 대습 호주가산 규칙
  - 형제자매 상속 단계에서도 구법상 남녀/출가 차등이 다시 적용될 수 있다는 실무 규칙
  - `1969년 김주수 친상법 이론서`를 참고자료로 반영하되, 대습 호주가산은 후속 판례·등기실무 우선 원칙 명시
  - 본위상속/대습상속 구분
  - 엔진 디버깅 시 `calcSteps`, `mergeSources`, `sourceBreakdowns` 읽는 법

### 2. 마스터 인덱스 업데이트
- `!명세서/00_MASTER_INDEX.md`
  - 계산룰 문서 항목 추가

### 3. 엔진 출력 보강
- 파일:
  - `src/engine/inheritance.js`
- 추가 필드:
  - `calcSteps[].sourceBreakdowns`
  - `finalShares.subGroups[].sourceBreakdown`

### 4. 출처별 분배근거 보존
- 같은 인물에게 여러 출처에서 지분이 들어오는 경우:
  - `mergeSources`: 출처 요약
  - `sourceBreakdowns`: 출처별 법시기, 입력분수, 하위 분배 상세
  를 함께 보존하도록 보강

예:
- `김명남`
  - `김혁조분 1/15`
  - `구수명분 2/165`
  를 따로 확인 가능

### 5. 김명수-정문자 사례를 규칙 예시로 문서화
- `정문자`는 `김명수` 사망 당시 생존 상속인이므로 `김명수분`을 먼저 취득
- 그러나 `구수명` 사망(1990) 전인 1986년에 사망했으므로 `구수명분`에서는 제외
- `정문자` 재산의 재분배는 1986년 법구간(1979~1990)으로 보고
  - 남자 형제 `1`
  - 출가녀 `1/4`
  규칙을 다시 적용해야 함
- 따라서 `정문자` 몫을 형제자매에게 단순 `1/6`씩 균등 분배한 계산은 법정상속분 기준으로 오류 가능성이 큼

## 김혁조 사건 기준 확인 포인트

### 현재 확인된 점
- 루트 분배는 `2/15, 1/15, 1/15, 1/15, 6/15, 4/15` 구조로 정상화됨
- 가지 소계는
  - `김명남 13/165`
  - `김명수 13/165`
  - `김말수 13/165`
  - `김일상 74/165`
  - `김일주 52/165`
  로 맞아짐
- 전체 합계는 `1/1`

### 아직 검토 중인 점
- `김명남` 등 일부 가지의 말단 지분이 신청인표와 다름
- 이 차이는 단순 버그일 수도 있고, 신청인 계산의 법리 선택 차이일 수도 있음
- 특히 `대습 호주가산(선례 2-285, 8-187)` 적용 여부는 별도 검토 필요

## 운영 원칙
1. 신청인 계산표는 참고자료로 활용하되 절대 기준으로 고정하지 않는다.
2. 먼저 법리와 사망시점별 적용법을 확인한다.
3. 다음으로 엔진의 출처별 분배근거를 확인한다.
4. 그 후 신청인표와 비교해 차이 이유를 기록한다.
