# v4.49 불러오기 서브트리복제중단 재링크우선

## 배경
- `구수명 > 상속인 불러오기` 이후 메모리가 급격히 증가하고 결국 앱이 멈추는 현상이 재현되었다.

## 원인
- `appendResolvedHeirs()`가 기존 person을 재사용하지 않고, 불러오기 대상 자녀들의 서브트리를 새 personId로 통째로 다시 복제했다.
- 대형 가계에서는 이 과정이 중복 그래프를 대량 생성하여 메모리 폭증으로 이어졌다.

## 변경
1. 기존 `personId`가 `vault.persons`에 존재하면 해당 person을 재사용
2. 기존 person 재사용 시 하위 서브트리 재복제 중단
3. 같은 부모 아래 동일 대상 링크 중복 생성 방지

## 기대 결과
- `구수명` 같은 대형 탭에서 불러오기 시 메모리 급증 완화
- 동일인 정합성 개선
