# v4.50 스마트가이드 참조안정성 메모리누수원인 정리

## 요약
- 메모리 폭증의 한 핵심 원인은 `smartGuides` 배열과 `checkedGuideKeys` Set이 매 렌더마다 새 참조를 만들며 무한 리렌더 루프를 유발한 데 있었다.

## 핵심 원인
1. `smartGuides`를 매 렌더마다 새 배열로 생성
2. `useEffect([smartGuides])`가 매번 다시 실행
3. `setCheckedGuideKeys(new Set(...))`가 같은 내용이어도 새 참조를 반환
4. React가 상태 변경으로 판단하여 다시 리렌더

## 교훈
- 이 프로젝트처럼 대형 트리와 가이드를 다루는 화면에서는
  - 배열/객체/Set/Map의 참조 안정성이 메모리 안정성과 직결된다.

## 후속 지침
- 이후 새로운 실시간 가이드/검수 로직을 넣을 때는
  - `useMemo`
  - bail-out (`prev` 반환)
  - 모듈 레벨 상수 반환
를 먼저 점검한다.
