# v4.33 선사망 대습상속 자동복구 및 실시간 동기화 보강

## 1. 변경 배경

김혁조 사건 JSON을 불러온 뒤 `구수명` 측 계보를 확인하는 과정에서,
`김명남`, `김명수`, `김말수`처럼 **선사망이지만 하위 상속인이 이미 입력된 사람들**이
여전히 `상속권 없음`처럼 보이는 문제가 발견되었다.

이 현상은 계산 엔진 전처리와 UI 표시용 트리 재구성 로직이 서로 다르게 동작했기 때문에 발생했다.

- 계산 전처리: 하위 상속인이 있으면 제외 상태를 해제하려는 방향이 일부 반영되어 있었음
- UI 재구성: 선사망이면 일괄 `predeceased`로 처리하는 구간이 남아 있었음

그 결과 사용자는
`대습상속으로 내려가는 상태`를
`완전히 상속권이 없는 상태`로 오해하게 되었다.

---

## 2. 이번 버전의 핵심 판정 원칙

### 2.1. 선사망 자녀/형제자매

- `사망일 < 기준 사망일`인 경우 선사망으로 판정한다.
- 다만 **하위 상속인이 1명 이상 존재하면**, 더 이상 단순 `상속권 없음`으로 보지 않는다.
- 이 경우 시스템은 자동으로:
  - `isExcluded = false`
  - `exclusionOption = ''`
  로 복구하고,
  UI상태도 `대습상속 진행`으로 해석한다.

### 2.2. 선사망 + 하위 상속인 없음

- 선사망 상태이면서 하위 상속인이 없으면
  `isExcluded = true`
  `exclusionOption = 'predeceased'`
  를 유지한다.
- 이 경우에만 `상속권 없음 (선사망)`이 맞다.

### 2.3. 선사망 배우자

- 선사망 배우자는 이번 자동복구 규칙의 대상이 아니다.
- 기본적으로 OFF 상태를 유지한다.

## 3. 코드 반영 지점

### 3.1. `src/utils/treeDomain.js`

입력값 정규화 단계에서 선사망 자녀/형제자매를 처리할 때,
하위 상속인 수를 함께 확인하도록 수정했다.

- 하위 상속인 있음 → 기본 ON 복구
- 하위 상속인 없음 → `predeceased` 유지

### 3.2. `src/utils/vaultTransforms.js`

Vault에서 트리를 다시 빌드할 때도 동일한 규칙을 적용했다.
즉, **불러오기 직후 UI 상태와 계산 상태가 다르게 보이지 않도록** 맞췄다.

### 3.3. `src/utils/vault.js`

구형/보조 변환 경로에도 같은 기준을 반영해,
향후 다른 경로에서 동일한 불일치가 다시 나타나지 않도록 보강했다.

### 3.4. `src/components/HeirRow.jsx`

표시 문구를 다음 기준으로 보정했다.

- 선사망 + 하위 상속인 있음 → `대습상속 진행`
- 선사망 + 하위 상속인 없음 → `상속권 없음 (선사망)`
- 선사망 경로 버튼 → `대습상속 >>`

---

## 4. 실무상 의미

이번 변경의 목적은 단순히 문구를 예쁘게 바꾸는 것이 아니다.

핵심은 다음과 같다.

1. **입력 순서가 달라도 최종 판정이 같아야 한다.**
2. **하위 상속인이 추가되면 즉시 상태가 재해석되어야 한다.**
3. **UI가 법적 상태를 잘못 설명하면 안 된다.**

즉,
`선사망 본인 직접 상속 불가`
와
`대습상속 경로 활성`
는 같은 의미가 아니므로,
동일한 `상속권 없음` 문구로 묶지 않도록 한 것이다.

## 5. 확인 포인트

김혁조 사건 JSON 기준으로 다음을 다시 확인한다.

- `구수명` 탭 진입 시
  - `김명남`
  - `김명수`
  - `김말수`
  가 `상속권 없음`이 아니라 `대습상속 진행` 또는 ON 상태로 보이는지
- `김말남`은 하위 상속인이 없고 명시적 제외 상태이므로 기존처럼 제외로 유지되는지
- 하위 상속인을 추가/삭제했을 때 토글 상태와 가이드 문구가 즉시 바뀌는지

추가로 다음도 함께 확인한다.

- 선사망 배우자(예: 조흥이)는 이미 `상속권 없음`으로 정상 정리된 경우,
  별도의 필수 경고를 계속 띄우지 않는지
- 배우자 중복 경고는 **실제 상속 가능한 배우자**가 2명 이상일 때만 발생하는지
- 선사망으로 이미 제외된 배우자는 `배우자 중복` 논리 충돌 검사에서 제외되는지

---

## 6. Gemini 전달용 요약

이번 버전(v4.33)에서는
**선사망자에게 하위 상속인이 입력되면 자동으로 대습상속 경로를 활성화**하도록 고쳤다.

이제 시스템은 선사망자를 무조건 `상속권 없음`으로 보지 않고,

- 하위 상속인 없음 → `상속권 없음 (선사망)`
- 하위 상속인 있음 → `대습상속 진행`

으로 구분한다.

판정은 `treeDomain`, `vaultTransforms`, `vault`, `HeirRow`까지 함께 맞췄다.
