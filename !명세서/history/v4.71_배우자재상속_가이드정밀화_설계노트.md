# v4.71 배우자 재상속 가이드 정밀화 설계 노트

작성일: 2026-04-23
업데이트 반영: 2026-04-24

---

## 배경

이흥춘 사건 검토 과정에서 배우자 재상속 사건 안내가 실제 법리와 사용자 인식 모두에 어긋나는 문제가 확인됐다.

대표 문제:

- 노금례, 정순화 같은 사망한 배우자 사건에 대해
- 스마트 가이드 또는 엔진 경고가 `형제자매(차순위)에게 자동 분배`처럼 보이는 문구를 그대로 노출함
- 처 재상속 사건과 남편 재상속 사건을 같은 방식으로 안내함
- 현재 화면이 보여주는 정보와 비교 기준 정보를 혼동하는 문구가 섞여 있음

이 문제는 엔진 규칙 자체보다 `가이드 판정 방식`, `입력창 설명 방식`, `배우자 사건별 질문 순서` 문제에 가깝다.

---

## 이번 반영 방향

### 1. 사건 유형을 분리해서 본다

배우자 재상속은 하나의 흐름으로 묶지 않는다.

- 처 재상속 사건
- 남편 재상속 사건

두 경우를 별도 규칙으로 안내한다.

### 2. 처 재상속은 기본 확정형으로 본다

1991년 이전 사건에서는 계모자관계에 따라 남편 자녀가 처 사건의 기본 상속인으로 반영된다는 점을 가이드에서 분명히 말한다.

반영 원칙:

- `반영될 수 있습니다`가 아니라 `반영됩니다`로 안내
- 사용자는 필요하면 불러오기 후 수정하거나 제외할 수 있음
- 가이드는 기본 규칙을 분명히 말하고, 특수상황 조정은 사용자 입력에 맡김

### 3. 남편 재상속은 비교형으로 본다

남편 재상속에서는 상위 사건 자녀 목록을 비교 기준으로 보여주고,

- 일부 제외
- 전부 제외

가 가능하도록 본다.

다만 `전부 제외`는 자동으로 직계존속 또는 형제자매 단계로 확정하지 않는다. 차순위 상속인 입력이 필요한지 다시 안내하는 흐름으로 본다.

### 4. 계모자관계 안내는 오류 지적이 아니라 법리 안내로 쓴다

기존처럼 `상속 누락인지 확인해 주세요`라고 쓰면 오해의 소지가 있다.

채택 문구 방향:

- `1991년 이전에는 계모자관계에 따라 상속인에 포함될 수 있으니 확인 바랍니다.`

즉, 오류 단정이 아니라 법리에 따른 포함 가능성을 안내한다.

### 5. 스마트 가이드는 짧게, 기본 설명은 입력창에 둔다

스마트 가이드 역할:

- 짧은 행동 지시
- 사건 이동 또는 확인 유도

입력창 기본 정보 영역 역할:

- 상위 사건 자녀 목록 표시
- 비교 기준 설명
- 추가 입력 또는 제외 기준 설명

### 6. 버튼 문구는 유지한다

`불러오기` 버튼 문구는 유지한다.

이유:

- 버튼 폭 증가와 공간 낭비를 피한다.
- 대신 기본 정보창에서 무엇을 기준으로 봐야 하는지 먼저 안내한다.

## 이번 논의로 확정된 문구 방향

### 처 재상속

- `위 자녀들은 [이성우] 사건 기준 자녀 목록입니다.`
- `1991년 이전 사건에서는 이 자녀들이 [노금례] 사건의 기본 상속인으로 반영됩니다.`
- `[노금례]에게만 있는 별도의 자녀가 있으면 추가로 입력해 주세요.`

스마트 가이드:

- `[노금례]에게만 있는 추가 자녀 여부를 확인해 주세요.`

### 남편 재상속

- `위 자녀들은 [처 이름] 사건 기준 자녀 목록입니다.`
- `[남편 이름] 사건에서는 이 목록이 그대로 상속인이 아닐 수 있습니다.`
- `상속인이 아닌 사람이 있으면 선택해 주세요.`

스마트 가이드:

- `[남편 이름] 사건의 자녀 범위를 확인해 주세요.`

### 1991년 이전 계모자관계 누락 감지

- `[홍길동]은(는) [남편 이름]의 자녀로 입력되어 있습니다. 1991년 이전에는 계모자관계에 따라 [처 이름] 사건의 상속인에 포함될 수 있으니 확인 바랍니다.`

복수형:

- `1991년 이전 사건입니다. [처 이름] 사건에 포함되지 않은 [남편 이름]의 자녀가 있습니다: [A], [B]. 계모자관계에 따라 상속인에 포함될 수 있으니 확인 바랍니다.`

---

## 남은 구현 과제

1. 배우자 재상속 사건 입력창에 상위 사건 자녀 목록을 표시할 위치 확정
2. 남편 재상속 사건의 일부 제외 / 전부 제외 UI 설계
3. 전부 제외 후 차순위 상속인 입력 유도 문구 설계
4. 1991년 이전 계모자관계 누락 감지 비교 규칙 구현
5. 스마트 가이드와 입력창 설명이 서로 같은 내용을 중복 출력하지 않도록 정리

---

## 결론

이번 설계 정리는 `배우자 재상속 안내를 더 편리하게 만든다` 수준을 넘어서, 사건 유형별 법리 차이를 사용자에게 덜 헷갈리게 전달하기 위한 기준을 세운 것이다.

핵심 결론은 다음과 같다.

- 처 재상속은 기본 확정형
- 남편 재상속은 비교형
- 계모자관계 안내는 법리 안내형
- 스마트 가이드는 짧게, 기본 설명은 입력창에 배치
- 버튼 문구는 유지하고, 비교 기준은 기본 정보창에서 설명
