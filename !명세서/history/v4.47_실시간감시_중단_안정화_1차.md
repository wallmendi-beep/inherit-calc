# v4.47 실시간감시 중단 안정화 1차

## 배경
- 메모리 사용량이 시간이 지날수록 급증하여 최대 9GB 이상까지 상승하고 앱이 멈추는 현상이 확인되었다.
- 최근 추가된 실시간 검수/가이드 갱신 로직이 주요 원인 후보로 판단되었다.

## 이번 변경
1. 입력 중 `importIssues` 실시간 재검사 중단
2. 스마트 가이드는 `input` 탭 또는 가이드 패널이 실제로 열려 있을 때만 계산

## 의도
- 실시간 감시 루프를 끊어 메모리 상승을 안정화한다.
- 당장 안정성을 확보한 뒤, 이후 수동 검수/반영 버튼 구조로 확장할 수 있게 한다.

## 주의
- import 검수 항목은 불러오기 시점 기준으로 유지된다.
- 입력 중 즉시 자동 정정/자동 축소는 줄어들 수 있으나, 현재 단계에서는 안정성을 우선한다.
