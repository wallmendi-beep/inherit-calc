**수정 파일:** `src/engine/inheritance.js`  
**심각도:** 🔴 Critical (산출 결과 오류 — 실무 반영 불가 수준)

## 1. 발견된 오류 현상

1979년 이전 구 민법이 적용되는 망인(1979~1990년 사망)의 상속 단계에서, **비고란의 설명과 실제 계산 분수가 불일치**하는 현상이 발견되었다.

**오류 재현 조건**  
망인이 복수의 경로로 상속을 받는 구조, 예: 피상속인(조부 등)에게서 직접 상속 + 사망한 처(妻) 경유 재상속이 동시에 발생하는 경우.

**오류 출력 예시 (STEP 9 · 망 김순범, 1990년 사망)**

| 상속인 | 오류 분수 | 비고란 출력 | 올바른 분수 |
|---|---|---|---|
| 강여수 (처) | 3/17 | 처(배우자) 5할 가산 | **6/17** |
| 김명조 (딸, 출가녀) | 2/17 | 출가녀 감산 (남자의 1/4) | **1/17** |
| 김용부 (아들, 호주) | 2/17 | 호주상속 5할 가산 | **6/17** |
| 김철헌 (아들) | 2/17 | — | **4/17** |

비고란은 "출가녀 감산", "호주상속 5할 가산"을 올바르게 식별하면서도, 실제 분수는 가중치가 전혀 반영되지 않은 **1991년 민법 기준 균등 배분** 값으로 잘못 계산되었다.

## 2. 근본 원인 분석

### 원인: Step 병합(Merge) 단계의 `h.r` 참조값 오염

버그는 입력 데이터가 아닌 **계산 엔진의 Step 병합 로직**에 존재한다.

**실행 흐름:**

```
① 1차 traverse (피상속인 → 고춘녀(처) → 김순범)
   → h.r 정상 설정: 출가녀=0.25, 호주아들=1.5, 처=1.5
   → 김순범 Step을 steps[] 에 push (sn/sd 올바르게 계산됨)

② 재귀 traverse (강여수 분배, 2010년 사망)
   → 1991년 민법 적용 → 강여수의 자녀(= 김순범의 자녀와 동일 객체 참조)
   → h.r을 1.0으로 덮어씀  ← 객체 참조 공유로 인한 오염 발생

③ 2차 traverse (피상속인 → 김순범 직접 경로)
   → 김순범 Step이 두 번째로 생성 → Merge 대상

④ Merge 단계 (lines 541-549)
   → d.h.r 을 참조하여 sn/sd 재계산
   → 이미 오염된 h.r = 1.0 이 사용됨
   → 모든 자녀가 균등 비율(2/17 등)로 잘못 산출

⑤ d.mod (비고) 는 스냅샷 값이므로 "출가녀 감산" 텍스트가 그대로 남음
   → 비고와 분수가 모순되는 출력 발생
```

**핵심:** `d.h` 는 트리 원본 객체의 **참조(Reference)** 이다. 강여수의 traverse가 동일 객체의 `h.r`을 1991 민법 기준으로 덮어쓴 이후, Merge 단계에서 이 오염된 값이 재계산에 사용된다.

## 3. 수정 내용

### 수정 방법: 계산 시점의 `h.r` 스냅샷(`rSnap`) 저장

`h.r`은 이후 traverse에 의해 변경될 수 있는 가변값이므로, **dist 저장 시점에 `rSnap` 필드에 스냅샷**을 함께 저장하고, Merge 단계에서 `d.h.r` 대신 `d.rSnap`을 사용하도록 수정했다.

**수정 위치 1 — dist 저장 시 rSnap 추가** (`~line 497`):

```js
// 수정 전
step.dists.push({ h, n: nn, d: nd, sn, sd, mod: h.modifierReason });

// 수정 후
step.dists.push({ h, n: nn, d: nd, sn, sd, rSnap: h.r, mod: h.modifierReason });
//                                              ↑ 계산 시점 h.r 스냅샷 보존
```

**수정 위치 2 — Merge 시 rSnap 사용** (`~line 541`):

```js
// 수정 전
const total = existing.dists.reduce((sum, d) => sum + (d.h?.r || 0), 0);
if (total > 0) {
  existing.dists = existing.dists.map(d => {
    if (d.h?.r === 0 || d.h?.r === undefined) return { ...d, n: 0, d: 1, sn: 0, sd: 1 };
    const [sn, sd] = math.simplify(d.h.r * 100, total * 100);
    ...
  });
}

// 수정 후
const total = existing.dists.reduce((sum, d) => sum + (d.rSnap ?? d.h?.r ?? 0), 0);
if (total > 0) {
  existing.dists = existing.dists.map(d => {
    const r = d.rSnap ?? d.h?.r ?? 0;
    if (r === 0) return { ...d, n: 0, d: 1, sn: 0, sd: 1 };
    const [sn, sd] = math.simplify(r * 100, total * 100);
    ...
  });
}
```

## 5. 영향 범위

**영향을 받는 케이스:**  
- 망인이 **2개 이상의 경로**(직접 + 처/부 경유)로 상속을 받는 구조
- 해당 망인의 사망일이 **1979~1990년 사이** (1979년 개정 민법 적용 구간)
- 망인의 상속인 중 **출가녀** 또는 **호주상속인**이 포함된 경우

**영향을 받지 않는 케이스:**  
- 1991년 이후 사망 (개정 민법 균등 배분 → sn/sd 재계산 결과가 동일)
- 단일 경로 상속 (Merge가 발생하지 않으므로 오염 없음)
- 1960년 구 민법 구간도 동일 버그에 노출되어 있으므로 함께 수정됨
