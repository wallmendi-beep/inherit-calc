# v4.72 개인정보보호 저장방식 검토 노트

## 개요

사건 저장 파일의 개인정보 노출 가능성을 줄이기 위해 저장 포맷 보호 방안을 검토했다.

이번 단계에서는 코드를 즉시 도입하지 않고, 요구사항과 구현 제약을 문서화하는 데 집중했다.

## 정리한 내용

- 메모장 즉시 열람 방지용 `전용 포맷 + 난독화` 방식
- 프로그램 내장 키 기반 자동 암호화 방식
- OS 계정 종속 자동 복호화 방식
- 사용자 비밀번호 기반 암호화 방식

## 핵심 결론

- 비밀번호 없이 자동으로 열리게 하면서 강한 보안을 제공하는 것은 어렵다
- 단기적으로는 `전용 포맷/난독화`가 가장 구현이 쉽다
- 실질적인 개인정보 보호를 원하면 장기적으로 `OS 계정 종속 암호화`가 더 적합하다
- 내장 키 방식은 보안 기대 대비 실제 강도가 낮아 최종안으로는 신중해야 한다
