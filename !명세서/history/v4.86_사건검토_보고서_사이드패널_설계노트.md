## 1. 배경

사건 검토 탭은 기존 사건 그래프와 관계 트리의 한계를 보완하기 위해 상속지분 마인드맵 방식으로 재설계 중이다.

이 과정에서 사건 보고서가 다음 문제를 보였다.

- 노드 설명과 판정 문서의 역할이 섞임
- 원자료 전체, 취득경로, 가지 요약 등 클릭한 사건과 무관한 정보가 섞임
- `판정`, `가감산 요소`, `재상속/대습상속` 표기가 중복됨
- 적용법 표기가 `79'민법`, `1979년 민법`, `1979년 개정민법` 등으로 흔들림
- 상속권 없는 사람은 그래프 노드에는 나오지 않아야 하지만, 보고서에는 이유가 표시되어야 함

---

## 2. 설계 결론

사건 검토 탭은 다음 구조로 정리한다.

| 영역 | 역할 |
|---|---|
| 좌측 사이드패널 | 선택한 노드가 속한 사건의 사건 보고서 |
| 우측 메인 영역 | 상속지분 마인드맵 |

`데이터 입력` 탭에서는 좌측 사이드패널이 상속가계도이고, `사건 검토` 탭에서는 같은 위치의 사이드패널이 사건 보고서가 된다.

보고서는 노드를 해설하는 화면이 아니라, **클릭한 상속사건 하나에 대한 판정 문서**다.

---

## 3. 보고서 표시 원칙

### 3.1 상단 기본정보

사이드패널 문서임을 알 수 있도록 `사건 보고서` 제목을 두고, 기본정보는 두 줄로 압축한다.

```text
사건 보고서
피상속인 김명수   상속지분 1분의 1
사망일자 1967년 10월 27일   적용법 구민법
```

적용법 표기는 다음 3개로 고정한다.

- `구민법`
- `79년 민법`
- `현행민법`

### 3.2 사건 관계인

표 컬럼은 다음 4개만 둔다.

```text
성명 / 관계 / 지분 / 가감산 요소
```

원칙:

- `판정` 컬럼은 두지 않는다.
- `상속권 없음`은 `가감산 요소` 칸에 표시한다.
- 재상속/대습상속은 가감산 요소가 아니므로 이 칸에 넣지 않는다.
- 지분은 가능한 한 통분지분으로 표시하되, 사건 관계인 표에서는 `2/165`처럼 분수로 표시한다.
- 상속권 없는 사람은 그래프 노드에서는 제외하되, 보고서의 사건 관계인에는 표시한다.

### 3.3 사건 요약

사건 요약은 다음 3개 하위 섹션으로 분리한다.

```text
상속권 없음
가감산 요소
재상속·대습상속
```

한 사람이 여러 사유를 가지면 섹션별로 각각 표시한다.  
예를 들어 재상속을 받는 사람이 가감산 사유도 있으면 두 섹션에 모두 표시한다.

문구는 `[이름] + 날짜/사유 + 결과` 형식으로 짧게 유지한다.

```text
상속권 없음
- [정문자] 1962년 05월 10일 혼인으로 상속권 없음
- [정문자] 1968년 02월 03일 재혼으로 상속권 없음

가감산 요소
- [정옥재] 1965년 03월 12일 출가로 감산
- [정인숙] 출가로 감산
- [김철수] 호주상속으로 가산

재상속·대습상속
- [구수명] 1979년 00월 00일 사망으로 재상속
- [정문자] 0000년 00월 00일 사망으로 대습상속
```

날짜가 없으면 날짜를 생략한다. `날짜 미상`은 반복 표기하지 않는다.

### 3.4 판정 요약

표와 사건 요약만으로 부족한 경우에만 짧게 보충한다.

금지:

- 노드 그래프 설명
- 원자료 전체 목록
- 취득경로 요약
- 가지 요약
- 같은 내용을 표와 요약에서 장문으로 반복

## 4. 코드 수정 계획

### 4.1 데이터 매핑

- 선택 노드 기준으로 해당 `calcStep`을 찾는다.
- 보고서에는 해당 사건의 관계인만 표시한다.
- 그래프에서 숨긴 `상속권 없음` 대상도 보고서 데이터에는 포함한다.
- raw 입력값에서 사망일, 출가일, 혼인일, 재혼일, 가적 상태, 호주상속 여부를 조회해 요약 문구에 반영한다.

### 4.2 적용법 표기 통일

공통 포맷터를 둔다.

```text
구민법
79년 민법
현행민법
```

보고서, 노드 보조표기, 배지, 요약 문구에서 같은 포맷터를 사용한다.

### 4.3 사건 관계인 표

현재 보고서 표를 다음 컬럼으로 축소한다.

```text
성명 / 관계 / 지분 / 가감산 요소
```

`판정` 컬럼은 제거한다.  
상속권 없음은 `가감산 요소`에 표시한다.
사건 관계인 표의 지분은 `2/165`처럼 분수로 표시한다.

### 4.4 사건 요약 생성기

사건 요약은 세 배열로 분리 생성한다.

- `상속권 없음`
- `가감산 요소`
- `재상속·대습상속`

각 항목은 raw 입력값과 엔진 판정값을 조합해 짧은 문구로 만든다.

### 4.5 레이아웃

- `tree` 탭 좌측 사이드패널을 사건 보고서로 사용한다.
- 메인 영역은 상속지분 마인드맵이 최대한 넓게 사용한다.
- 보고서는 좁은 세로 문서로 설계하고, 헤더 카드나 큰 기본정보 박스를 두지 않는다.

## 7. 후속 수정 메모

### 7.1 사건 관계인 표 — 아들 가감산 요소 공란 처리

수정 원칙:

- 아들은 동일가적 여부 정보를 수집하지 않는다.
- 따라서 아들 행에서 가감산 사유가 없으면 `동일가적`을 표시하지 않는다.
- `아들 + 가감산 없음`은 공란 처리한다.
- 딸은 동일가적/비동일가적, 출가감산 등 기존 판단을 유지한다.

구현 상태:

- `src/components/TreePanel.jsx`의 `formatModifierShort(dist)`에 반영.
- 현재 UI에서는 공란 대신 표 구조 유지를 위해 `-`로 표시한다.

### 7.2 사건 보고서 — raw 기준 관계인 병합

상속제외 대상자 중 일부는 엔진의 `isRenounced()` 필터에서 먼저 제거되어 `step.dists`에 들어오지 않는다.  
그 결과 사건 보고서에도 표시되지 않는 문제가 있었다.

수정 원칙:

- 그래프 노드는 계산 결과 기준으로 유지한다.
- 사건 보고서는 `raw 사건 관계인 + 엔진 판정 결과` 기준으로 만든다.
- `step.dists`에 없는 raw 관계인 중 상속제외, 혼인/재혼, 선사망, 대습불가 등으로 빠진 사람은 보고서에 `상속권 없음`으로 표시한다.
- 계산용 `targetHeirs`와 보고서용 `casePersons`를 분리한다.

구현 상태:

- `src/components/TreePanel.jsx`에 `buildReportRows(reportNode, eventDate)` 추가.
- `step.dists`와 `reportNode.step.dec.heirs`를 병합한다.
- `renounce`, `remarried`, `blocked_husband_substitution`, `lost`, `disqualified`, 이혼/재혼 자동 제외 등은 보고서 전용 `상속권 없음` 행으로 표시한다.

### 7.3 마인드맵 노드 지분 — 전역 통분 금지

마인드맵 노드 카드 지분을 그래프 전체 분모로 통분하면 `170270100` 같은 거대한 분모가 발생한다.  
이는 비교를 쉽게 하려는 목적에 반한다.

수정 원칙:

- 엣지는 사건 내부 분배분수(`sn/sd`)를 같은 부모 사건의 형제 그룹 기준으로 통분해 표시한다.
- 노드 카드 지분은 전역 통분하지 않는다.
- 통분이 필요한 경우 **같은 부모 아래의 형제 노드끼리만** 통분한다.
- 부모 재상속 노드와 그 자식 사건의 취득 노드를 억지로 같은 분모로 맞추지 않는다.
- 거대 분모가 생기는 방식의 전체 그래프 최소공배수 통분은 금지한다.

예시:

```text
김혁조 사건 형제 노드
구수명 2/15
김명남 1/15
김명수 1/15
김말수 1/15
김일상 6/15
김일주 4/15

구수명 하위 사건
김경진 33/1155
김화경 33/1155
김지훈 33/1155
김진수 33/1155
```

구현 상태:

- `src/components/TreePanel.jsx`의 전역 `graphDenom` 방식을 제거했다.
- 부모 사건의 형제 노드 그룹 기준 `cardDenom`만 사용한다.
- 재상속/대습상속 이벤트 노드는 부모 사건에서 받은 지분을 형제 그룹 기준으로 표시한다.
- 엣지는 계속 사건 내부 분배분수(`sn/sd`)를 표시하되, 같은 부모 사건의 형제 그룹 안에서는 비교 가능하도록 통분한다.

## 8. 구현 요약

2026-05-04 구현 완료 사항:

- 사건 검토 탭을 상속지분 마인드맵 중심으로 전환
- 좌측 사건 보고서 사이드패널 도입
- 사건 보고서 기본정보 2줄 압축
- 적용법 표기 `구민법 / 79년 민법 / 현행민법` 통일
- 사건 관계인 표 `성명 / 관계 / 지분 / 가감산 요소`로 축소
- 상속권 없음 대상자는 그래프에서 제외하고 보고서에 표시
- 사건 요약을 `상속권 없음 / 가감산 요소 / 재상속·대습상속`으로 분리
- 노드 클릭 시 선택 노드가 속한 사건 보고서를 표시
- 노드 카드 크기와 표시정보를 줄여 분기 구조 중심으로 정리
- 재상속/대습상속 이벤트 노드에도 관계 표기
- 가계도 요약 패널을 데이터 입력 탭 전용 좌측 패널로 정리
- 사건 검토 탭에서는 가계도 요약 패널 렌더링 차단
