# 🎨 상속지분 계산기 PRO - UI & 통합 디자인 시스템 (UDS)

본 문서는 앱의 시각적 일관성과 '전문적 신뢰감'을 유지하기 위한 **디자인 최종 사양서**입니다. Notion 스타일의 미니멀리즘과 전문가용 정밀 레이아웃을 결합한 규격을 정의합니다.

---

## 1. 디자인 정체성 (Identity)

*   **스타일**: **Notion-inspired Professional Minimalist**
*   **핵심 원칙**: 
    - 화려한 색상보다 종이 질감의 무채색과 정갈한 타이포그래피 우선.
    - 화려한 애니메이션보다 데이터의 선명도와 연산의 정확도 우선.
    - 모든 조작계는 일관된 간격과 피드백을 제공(전문가용 도구).

---

## 2. 컬러 및 타이포그래피 (Tokens)

### 2.1. 컬러 팔레트
| 구분 | HEX 코드 | 명칭 | 주요 용도 |
| :--- | :--- | :--- | :--- |
| **Black** | `#37352f` | Notion Black | 주요 텍스트명, 피상속인 성명, 입력값 |
| **Gray** | `#787774` | Notion Gray | 보조 설명, 미 활성 레이블, 비활성 텍스트 |
| **Navy** | `#1e56a0` | Deep Navy | **핵심 지분 데이터(N/D)**, 활성 강조 포인트 |
| **Blue** | `#0284c7` | Sky Blue | 액션 버튼, 자동 판별 상태 스타일 |
| **Red** | `#c93f3a` | Muted Red | 오류, 상속권 상실, 감산 가중치 뱃지 |

### 2.2. 타이포그래피 계층
- **L1 (Heading)**: `17px / Bold / #37352f` (모달 제목, 섹션 제목)
- **L2 (Body)**: `15px / Medium / #37352f` (성명, 관계 등 주요 데이터)
- **L3 (Label)**: `13px / Semibold / #787774` (입력 필드 네이밍, 설명)
- **L4 (Data)**: `11px / Black / #1e56a0` (지분율 N/D 수치 - 가시성 극대화)

## 3. 물리적 레이아웃 규격 (Dimensions)

### 3.1. 메인 덱 (Main Deck)
- **본문 고정 너비**: `w-[1080px]` (가이드 패널 오픈 시에도 유지되도록 `shrink-0` 적용)
- **카드 내부 여백**: `p-10`
- **모서리 곡률**: `rounded-xl`
- **테두리**: `border border-[#e9e9e7]` (가느다란 실선 스타일)

### 3.2. 상속인 행(HeirRow) 수평 정렬 여백 (Fixed Margins)
- **드래그 핸들**: `ml-[10px]`
- **상태 토글**: `ml-[20px]`
- **성명 입력**: `ml-[50px]`
- **관계 선택**: `ml-[30px]`
- **사망여부**: `ml-[30px]`
- **특수조건**: `ml-[10px]`
- **삭제 버튼**: `ml-[30px] mr-[20px]`

## 4. 원자적 컴포넌트 규격 및 시각적 박제 (Snapshots)

### 4.1. 뱃지 토글 (Badge Toggle)
- **원칙**: 슬라이더 방식이 아닌, **뱃지 자체가 클릭 시 반전**되는 캡슐 형태.
- **스타일**:
    - 일반: `border-[#e5e5e5] bg-[#f8f8f7] text-[#a3a3a3]` (비활성)
    - 활성: `border-neutral-400 bg-neutral-100 text-[#37352f]`
    - 자동판별: `border-blue-400 bg-blue-50/50 text-blue-700`

### 4.2. 가중치 뱃지 (Multiplier Badge)
- **형태**: **`[]` 사각형 형태** (`rounded-[3px]`) / 타원형(`rounded-full`) 지양.
- **가산(Bonus)**: `border-sky-800/80` / `text-sky-800/80` (예: 호주 x 1.5)
- **감산(Penalty)**: `border-rose-800/80` / `text-rose-800/80` (예: 딸 x 1/2)

### 4.3. 스마트 가이드 패널
- **너비**: `310px` 고정.
- **아이콘 사용 규칙**:
  - 섹션 헤더(예: `필수 검토`, `권장 가이드`, `참고사항`)에만 상태 이모지/아이콘을 **1회만** 사용한다.
  - 개별 카드마다 동일한 이모지/아이콘을 반복 노출하지 않는다.
  - 카드 내부는 텍스트 위계와 배경/테두리 톤 차이로만 구분한다.
- **문구 길이 규칙**:
  - 카드 본문은 가능한 한 **1문장, 40자 내외**의 짧은 안내를 우선한다.
  - 원인과 조치를 함께 써야 할 때는 `현상 + 조치` 형태로 압축한다.
  - 장문 설명은 카드 본문 대신 툴팁, 세부 패널, 또는 클릭 후 위치 이동으로 보완한다.
- **테마**: 무채색(`bg-neutral-50`) 바탕의 절제된 디자인.

---

## 5. 인쇄 및 워터마크
- **워터마크**: 입력 데이터 패널 우측 하단 여백. `Caveat`, `Dancing Script` 등 필기체 서체 사용, `16px`. 라이트/다크 모드에 맞춰 `text-shadow`를 활용한 세련된 음각(Engraved) 효과를 적용.
- **인쇄 모드**: `no-print` 태그를 통한 불필요 UI(삭제 버튼 등) 제거.

---

## 6. 텍스트 경량화 원칙
- **결과표 취득 경로**: 조사와 불필요한 서술어를 제거하고 직관적으로 축약한다.
  - 기존: "망 김영서의 아들으로 1979 개정민법 적용"
  - 개선: "망 김영서의 아들 <1979년 개정민법>"
- **스마트 가이드 상태 문구**:
  - 정상 상태 문구는 직관적이고 긍정적인 단어("이상 없음")로 시작하며 줄바꿈을 활용하여 스캐너블하게 작성한다.
