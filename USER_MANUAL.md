# 📖 상속지분 계산기 PRO - 사용자 매뉴얼

> 최종 업데이트: 2026-04-29 (v1.1.0 기준)

---

## 1. 전체 화면 구성

상속지분 계산기 PRO는 복잡한 가계도를 쉽게 입력하고 검증할 수 있도록 직관적인 패널 구조를 채택하고 있습니다.

### 1.1. 상단 메뉴바 (Top Toolbar)
화면 최상단에 고정되어 있는 메뉴바는 프로젝트 관리 및 유틸리티 기능을 제공합니다.

* **프로젝트 정보**: 좌측에 현재 사건 번호와 피상속인 이름이 표시됩니다.
* **가계도 전체 AI 자동입력 (\*)**: 복잡한 사건 내용을 자연어 텍스트로 붙여넣으면 AI가 분석하여 한 번에 가계도를 구성해줍니다.
* **가이드 열기**: 현재 가계도에서 확인해야 할 필수/권장 조치사항 패널을 엽니다. 미해결 필수 항목이 있으면 아이콘이 주황색으로 강조됩니다.
* **이전/다시 (Undo/Redo)**: 실수로 지우거나 수정한 내역을 앞뒤로 되돌릴 수 있습니다.
* **초기화**: 현재 입력된 모든 데이터를 지우고 처음부터 다시 시작합니다.
* **불러오기 / 저장**: 작성된 가계도 데이터를 `.json` 파일로 저장하거나 불러옵니다.
* **CSV**: 최종 계산된 상속인 목록과 지분을 엑셀(CSV) 형식으로 내보냅니다.
* **인쇄**: 계산 결과, 요약표, 사건 그래프 등을 문서 레이아웃으로 출력하거나 PDF로 저장합니다.
* **화면 배율 (- / +)**: UI 전체 크기를 확대 또는 축소합니다.
* **다크모드 (해/달 아이콘)**: 라이트/다크 테마를 즉시 전환합니다.

---

### 1.2. 좌측 패널 (사이드바 트리)
가계도 구조와 상속 흐름을 시각적으로 파악하고 탐색하는 미니 맵 영역입니다.

* **미니 트리 뷰**: 전체 가족 구조를 계층형으로 보여줍니다. 이름 클릭 시 해당 상속인의 입력 화면으로 즉시 이동합니다.
* **이름 검색**: 사이드바 상단 검색창에서 이름으로 상속인을 빠르게 찾을 수 있습니다.

---

### 1.3. 우측 패널 (스마트 가이드)
입력 데이터의 논리적 오류를 실시간으로 감지하고 법률 쟁점을 안내하는 핵심 비서 역할을 합니다.

* **필수 항목 진행률**: 상단 진행 바에서 필수 항목 완료 비율(X/N)을 한눈에 확인합니다. 전부 완료되면 초록색으로 전환됩니다.
* **필수 검토 (우선 확인)**: 사망일자 누락, 관계 오류, 순환 참조 등 계산을 막는 치명적 오류를 표시합니다. 이 항목을 해결해야 정확한 지분 계산이 진행됩니다.
* **권장 가이드**: 호주상속 지정, 구법 여성 상속인 혼인 정보, 재상속 검토 등 법적 정확도를 높이기 위한 권장 사항을 안내합니다.
* **확인 완료 항목**: 검토를 마친 항목은 접이식 "확인 완료" 목록으로 이동합니다. 언제든 다시 펼쳐 확인하거나 해제할 수 있습니다.
* **이상 없음**: 모든 검증이 완료되면 "이상 없음" 메시지가 표시됩니다.

> **팁**: 권장 가이드 항목이 실제로 다른 사람인 경우(예: 동명이인) "확인 (다른 사람)" 버튼을 눌러 완료 처리할 수 있습니다.

## 2. 날짜 입력 안내

모든 날짜 입력 필드는 `YYYY-MM-DD` 형식을 사용합니다.

* **자동 포맷**: 숫자만 입력해도 자동으로 구분자(`-`)가 삽입됩니다.
* **유효성 검사**: 존재하지 않는 날짜(예: `2024-02-30`)나 미완성 입력은 포커스가 벗어날 때 빨간 테두리와 "올바른 날짜가 아닙니다" 툴팁으로 표시되며 저장이 차단됩니다.
* **선사망 힌트**: 상속인 사망일이 피상속인 사망일보다 이른 경우 파란 테두리와 "선사망 처리" 힌트가 표시됩니다. 이 경우 대습상속 탭이 자동으로 생성됩니다.

## 3. 메인 탭(Tab)별 주요 기능

화면 중앙의 메인 작업 영역은 업무의 흐름에 따라 5개의 탭으로 나뉘어 있습니다.

### 📍 1단계: 데이터 입력
실제 가족 관계와 사망일자 등 기초 데이터를 입력하는 가장 중요한 화면입니다.

* **기본정보**: 최상단에서 사건번호, 피상속인 이름, 사망일자를 입력합니다. 구민법(1991년 이전 사망) 사건이면 호주 여부가 자동으로 표시됩니다.
* **사건 탭**: 재상속·대습상속이 발생하면 탭이 자동으로 생성됩니다. 탭을 클릭해 해당 사건의 상속인을 따로 입력할 수 있습니다.
* **빠른 등록**: 하단 입력창에 `김철수, 이영희` 처럼 이름을 쉼표로 구분해 입력하면 한 번에 여러 상속인이 등록됩니다.
* **개별 설정**: 각 상속인 행에서 관계, 생존/사망 여부, 사망일자, 호주 여부, 혼인·이혼·복적 날짜 등을 설정합니다.
* **관계 오류 경고**: 부모 성별이 바뀌어 현재 관계가 유효하지 않을 경우 빨간 텍스트와 "관계 재검토 필요" 툴팁이 표시됩니다.
* **불러오기**: 상위 사건의 자녀를 현재 탭으로 자동으로 불러올 수 있습니다.
* **없음 확정**: 더 이상 추가할 상속인이 없을 때 "없음 확정" 버튼을 눌러 가이드 경고를 해소합니다.

---

### 📍 2단계: 사건 검토
재상속·대습상속의 흐름을 시각적으로 검토하는 화면입니다.

* **사건 목록 & 다이얼**: 좌측 목록에서 여러 사건을 선택할 수 있습니다. 우측 다이얼을 드래그해 빠르게 이동할 수 있습니다.
* **뷰 전환**: 상단 버튼으로 "사건 그래프"(관계도)와 "관계 트리"(폴더형) 두 가지 뷰를 전환합니다.

#### 사건 그래프 읽는 법
각 사건별 카드에는 상속인 이름과 최종 지분만 표시됩니다. 계산 경위와 상속권 없음 사유는 카드 아래 **"이 사건 계산 근거"** 서술 블록에서 확인합니다.

| 항목 | 표시 내용 |
|---|---|
| **카드** | 이름 · 결과 지분 · 재상속/대습 버튼 |
| **서술 블록 - 취득자** | 이름(관계) · 사망일 · 계산 이유 · 대습/재상속 연결 여부 |
| **서술 블록 - 상속권 없음** | 취소선 이름 · 사망일 · 제외 이유 (선사망 후 대습 없음 명시) |

> **주의**: 선사망했다고 해서 무조건 대습상속이 개시되는 것은 아닙니다. 적격 대습상속인(배우자·직계비속)이 없는 경우 "적격 대습상속인 없어 상속권 소멸"로 표시됩니다.

---

### 📍 3단계: 계산 상세
법정 상속분이 어떤 수식으로 도출되었는지 단계별로 보여줍니다.

* **STEP별 표**: 각 상속 사건(피상속인)마다 상속인별 계산식과 결과가 표로 표시됩니다.
* **이름 클릭**: 상속인 이름을 클릭하면 데이터 입력 탭의 해당 인물로 이동합니다.
* **비고 컬럼**: 호주가산, 출가녀 감산 등 적용된 수정 사유가 표시됩니다.

---

### 📍 4단계: 지분 요약
최종 생존 상속인들의 통분 지분을 표로 요약합니다.

* **구조형/취득경로형** 두 가지 보기 방식을 전환할 수 있습니다.
* **인쇄** 버튼으로 즉시 보고서로 출력하거나 PDF로 저장할 수 있습니다.

---

### 📍 5단계: 구체적 상속분
법정 지분에 특별수익과 기여분을 반영한 구체적 상속분을 계산합니다.

* **재산 총액**: 상속 재산 총액을 입력합니다.
* **특별수익**: 각 상속인이 생전에 받은 증여액을 입력합니다 (간주상속재산에 가산됨).
* **기여분**: 각 상속인의 상속 재산 기여액을 입력합니다 (간주상속재산에서 차감됨).
* **최종 금액**: 위 항목을 반영한 각 상속인의 최종 취득 금액이 자동 계산됩니다.

## 4. 자주 묻는 질문 (FAQ)

**Q. 사건 탭이 자동으로 생기는 경우는?**  
상속인을 사망자로 입력하면 해당 인물의 대습상속 또는 재상속 탭이 자동으로 생성됩니다. 탭에서 해당 상속인의 후속 상속인을 입력하거나, "없음 확정"을 눌러 처리합니다.

**Q. 가이드의 "확인 (다른 사람)" 버튼은 언제 쓰나요?**  
동일 성명 중복 경고나 중복 인물 경고가 나타났을 때, 실제로 동명이인인 경우 "확인 (다른 사람)"을 눌러 경고를 완료 처리합니다. 완료 처리된 항목은 "확인 완료" 목록으로 이동하며 언제든 재확인할 수 있습니다.

**Q. CSV 내보내기는 어디서 하나요?**  
상단 메뉴바의 **CSV** 버튼을 클릭합니다. 성명·관계·법정지분·통분 지분·상속경로가 포함된 파일이 저장됩니다.

**Q. 저장 파일을 다른 PC에서 열 수 있나요?**  
네. 저장한 `.json` 파일을 다른 PC의 계산기에서 불러오기 하면 동일하게 불러올 수 있습니다. 모든 데이터는 파일 안에 포함됩니다.
